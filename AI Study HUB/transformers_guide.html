Attention Mechanism

Core concept · ~20 min read · Foundational

🔍 What Is Attention?

Attention is a mechanism that allows a model to focus on different parts of the input when producing each part of the output. Instead of compressing the whole input into a fixed vector, attention dynamically weights each input position.

Think of reading a sentence — you don't re-read every word to understand a pronoun; you attend to the relevant noun. Attention mirrors this selective focus computationally.

💡 Key insight: Attention replaced the bottleneck of RNN hidden states by allowing direct connections between any two positions in a sequence.

📐 Query, Key & Value

Every attention layer computes three projections of the input — Query (Q), Key (K), and Value (V) — using learned weight matrices.

Attention(Q, K, V) = softmax( QKᵀ / √dₖ ) · V

Q — "What am I looking for?"
K — "What do I contain?"
V — "What do I actually return?"

The dot product QKᵀ produces a score matrix. Dividing by √dₖ stabilises gradients. Softmax turns scores into a probability distribution, and V is then weighted-summed.

🔀 Multi-Head Attention

Instead of a single attention function, Transformers run h parallel attention heads, each learning different relationship types (syntax, coreference, semantics…). Outputs are concatenated and projected.

MultiHead(Q,K,V) = Concat(head₁,…,headₕ) · Wᴼ
where headᵢ = Attention(Q·Wᵢᵠ, K·Wᵢᴷ, V·Wᵢᵛ)
Python · PyTorch
import torch
import torch.nn as nn

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model=512, num_heads=8):
        super().__init__()
        self.d_head = d_model // num_heads
        self.num_heads = num_heads
        self.qkv = nn.Linear(d_model, d_model * 3)
        self.out = nn.Linear(d_model, d_model)

    def forward(self, x):
        B, T, C = x.shape
        qkv = self.qkv(x).chunk(3, dim=-1)   # split Q,K,V
        q, k, v = [t.view(B,T,self.num_heads,self.d_head)
                    .transpose(1,2) for t in qkv]
        scale = self.d_head ** -0.5
        attn = (q @ k.transpose(-2,-1)) * scale
        attn = attn.softmax(dim=-1)
        out = (attn @ v).transpose(1,2).contiguous()
        return self.out(out.view(B, T, -1))

⚡ Advanced: Flash Attention & Efficient Variants

Flash Attention 2 Sparse Attention Linear Attention Sliding Window Rotary Embeddings (RoPE) ALiBi

Standard attention is O(n²) in time and memory. FlashAttention rewrites the CUDA kernel to tile computations, achieving O(n) memory. Sparse attention (used in Longformer, BigBird) attends to only a subset of positions.

⚠️ For sequences > 4K tokens, always use an efficient attention variant — vanilla softmax attention will OOM on a 24 GB GPU at ~8K tokens with batch size 4.

BERT Architecture

Bidirectional Encoder · Google 2018 · Classification & Understanding

🏗️ What is BERT?

Bidirectional Encoder Representations from Transformers. BERT is an encoder-only model that processes the entire input sequence simultaneously — unlike GPT which processes left-to-right only.

BERT was pre-trained on two self-supervised tasks: Masked Language Modelling (MLM) and Next Sentence Prediction (NSP).

Token + Segment + Position Embeddings
↓
× 12 layers (BERT-Base) or 24 layers (BERT-Large)
Multi-Head Self-Attention (Bidirectional)
↓
Add & Norm → Feed-Forward → Add & Norm
↓
[CLS] token → Classification Head

🎭 Pre-Training Tasks

Masked LM: 15% of tokens are randomly masked. BERT learns to predict them using context from both directions.

Example — Masked LM
Input:  "The cat [MASK] on the mat"
Target: "The cat sat  on the mat"
# BERT sees ALL surrounding context → bidirectional

Next Sentence Prediction: Given sentence A and B, predict whether B follows A. Helps with tasks like QA and NLI.

✅ RoBERTa (2019) removed NSP, trained longer with more data and dynamic masking — significantly outperforming original BERT on most benchmarks.

🚀 Quick Start with BERT

Python · HuggingFace
from transformers import BertTokenizer, BertForSequenceClassification
import torch

tokenizer = BertTokenizer.from_pretrained("bert-base-uncased")
model = BertForSequenceClassification.from_pretrained(
    "bert-base-uncased", num_labels=2
)

text = "The transformer architecture revolutionised NLP."
inputs = tokenizer(text, return_tensors="pt", truncation=True)

with torch.no_grad():
    logits = model(**inputs).logits

pred = logits.argmax(dim=-1)
print("Predicted class:", pred.item())  # 0 = neg, 1 = pos

GPT Architecture

Decoder-Only · OpenAI · Autoregressive Text Generation

🧠 GPT vs BERT — Core Difference

GPT uses a decoder-only Transformer with causal (left-to-right) self-attention. At each position, the model can only attend to previous tokens. This makes GPT naturally suited for text generation.

⚙️ Modern GPT-style Improvements

Pre-LayerNorm: Apply layer norm before (not after) attention — more stable training.

Grouped Query Attention (GQA): Multiple query heads share key/value heads — reduces KV-cache memory by 4–8×.

SwiGLU: Replaces ReLU in FFN layers — empirically outperforms GELU on most benchmarks.

Fine-Tuning LLMs

Full FT · LoRA · QLoRA · RLHF · Instruction Tuning

🎯 Why Fine-Tune?

Pre-trained models have general knowledge but may lack domain expertise, output format control, or safety alignment. Fine-tuning adapts the model on task-specific data with a fraction of the original compute.

Full Fine-Tuning LoRA QLoRA Prefix Tuning Prompt Tuning RLHF DPO

🔧 LoRA — Low-Rank Adaptation

Instead of updating all weights (billions of params), LoRA injects small trainable rank-decomposition matrices into each layer. Only ~0.1–1% of parameters are trained.

W' = W₀ + ΔW = W₀ + B·A
where B ∈ ℝ^(d×r), A ∈ ℝ^(r×k), rank r << d,k
Python · PEFT + LoRA
from peft import LoraConfig, get_peft_model
from transformers import AutoModelForCausalLM

model = AutoModelForCausalLM.from_pretrained("meta-llama/Llama-3.2-3B")

lora_config = LoraConfig(
    r=16,               # rank
    lora_alpha=32,      # scaling factor
    target_modules=["q_proj", "v_proj"],
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM"
)

model = get_peft_model(model, lora_config)
model.print_trainable_parameters()
# trainable params: 4,194,304 || all params: 3,212,749,824 (0.13%)

🗜️ QLoRA — Quantised LoRA

QLoRA combines 4-bit quantisation of the base model with LoRA adapters trained in bf16. Enables fine-tuning a 65B model on a single 48 GB GPU.

Python · BitsAndBytes + QLoRA
from transformers import BitsAndBytesConfig
import torch

bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_quant_type="nf4",
    bnb_4bit_compute_dtype=torch.bfloat16,
    bnb_4bit_use_double_quant=True,   # nested quantisation
)

model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Llama-3.2-3B",
    quantization_config=bnb_config,
    device_map="auto"
)

📋 Full Fine-Tuning Workflow

1

Prepare Dataset

Format data as (prompt, completion) pairs. Use instruction format for chat models. Aim for 500–50K high-quality examples.

2

Choose Base Model

Pick a pre-trained checkpoint close to your domain. Smaller models (7B) train faster; larger (70B) generalise better.

3

Configure Training

Set learning rate (1e-4 to 2e-5 for LoRA), batch size, gradient accumulation, warmup steps, and max sequence length.

4

Train & Evaluate

Monitor train/eval loss. Use early stopping. Evaluate on held-out set with task-specific metrics (F1, ROUGE, accuracy).

5

Merge & Deploy

Merge LoRA weights into base model. Quantise for production (GGUF / AWQ). Serve with vLLM or HuggingFace TGI.

🏆 RLHF & DPO

RLHF (Reinforcement Learning from Human Feedback) trains a reward model from human preferences, then fine-tunes the LLM with PPO to maximise that reward — used in ChatGPT, Claude, Gemini.

DPO (Direct Preference Optimisation, 2023) simplifies RLHF by directly optimising on (chosen, rejected) pairs without needing a separate reward model. More stable and cheaper to run.

L_DPO(π) = -E[(x,y_w,y_l)] [ log σ( β·log(π(y_w|x)/π_ref(y_w|x)) − β·log(π(y_l|x)/π_ref(y_l|x)) ) ]

🏋️ Training with Trainer API

Python · Full Training Loop
from transformers import (
    AutoTokenizer, AutoModelForSequenceClassification,
    TrainingArguments, Trainer
)
from datasets import load_dataset
import numpy as np

# 1. Load data
ds = load_dataset("imdb")
tokenizer = AutoTokenizer.from_pretrained("distilbert-base-uncased")

# 2. Tokenize
def tokenize(batch):
    return tokenizer(batch["text"], truncation=True, padding=True)

ds = ds.map(tokenize, batched=True)

# 3. Load model
model = AutoModelForSequenceClassification.from_pretrained(
    "distilbert-base-uncased", num_labels=2
)

# 4. Training config
args = TrainingArguments(
    output_dir="./results",
    num_train_epochs=3,
    per_device_train_batch_size=16,
    learning_rate=2e-5,
    evaluation_strategy="epoch",
    save_strategy="epoch",
    load_best_model_at_end=True,
    fp16=True,
)

# 5. Train!
trainer = Trainer(model=model, args=args,
    train_dataset=ds["train"], eval_dataset=ds["test"])
trainer.train()

🎯 Interview Prep

Interview Q&A

30 questions · Beginner to Advanced · ML Engineer / Research roles
Beginner
Q01 What is the Transformer architecture and why was it a breakthrough?easy
+
The Transformer (Vaswani et al., 2017) is a model architecture based entirely on attention mechanisms, discarding recurrence and convolutions. It was a breakthrough because it enabled full parallelisation during training (unlike RNNs), captured long-range dependencies better, and scaled remarkably well with more compute and data. The self-attention mechanism allows every position to attend to every other position in a single layer, solving the vanishing gradient problem of RNNs for long sequences.
Q02 Explain the difference between encoder-only, decoder-only, and encoder-decoder models.easy
+
Encoder-only (e.g., BERT): Processes the full input bidirectionally. Best for understanding tasks — classification, NER, sentence similarity.

Decoder-only (e.g., GPT): Processes tokens left-to-right with causal masking. Best for generation tasks — completion, chat, code generation.

Encoder-Decoder (e.g., T5, BART): Encoder reads input, decoder generates output attending to encoder states. Best for seq2seq tasks — translation, summarisation, QA.
Q03 What is positional encoding and why is it needed?easy
+
Self-attention is permutation-invariant — it doesn't know the order of tokens. Positional encodings inject position information into the embeddings. Original Transformers used sinusoidal encodings: PE(pos, 2i) = sin(pos / 10000^(2i/d)), PE(pos, 2i+1) = cos(...). Modern LLMs prefer learnable relative encodings (RoPE, ALiBi) which generalise better to longer sequences than seen during training.
Q04 What is tokenisation? How does BPE work?easy
+
Tokenisation splits raw text into sub-word units. BPE (Byte-Pair Encoding) starts with individual characters, then iteratively merges the most frequent adjacent pair into a new token until a vocabulary size is reached. For example: "low", "lower", "newest" might share the token "low" and "est". This balances vocabulary size with coverage of rare words — unknown words are decomposed into known sub-words instead of becoming [UNK].
Intermediate
Q05 Why do we scale dot products by √dₖ in attention?mid
+
For large dₖ, dot products grow large in magnitude, pushing the softmax into saturation regions with very small gradients. Scaling by 1/√dₖ keeps the variance of the dot product at ~1 regardless of dₖ (assuming Q and K are unit-variance random vectors). This prevents vanishing gradients during backprop through softmax.
Q06 What is Layer Normalisation and how does it differ from Batch Norm?mid
+
Batch Norm normalises across the batch dimension — requires large batches and struggles with variable-length sequences. Layer Norm normalises across the feature dimension for each sample independently, making it batch-size invariant and well-suited for NLP. In Transformers, Pre-LayerNorm (applied before attention) is now preferred over the original Post-LayerNorm for training stability at scale.
Q07 What is the [CLS] token in BERT and how is it used?mid
+
BERT prepends a special [CLS] (classification) token to every input. After passing through all encoder layers, the [CLS] token's final hidden state aggregates information from the entire sequence via self-attention. This representation is used as the sequence-level embedding for downstream classification tasks. A linear head is added on top for fine-tuning.
Q08 What is the difference between fine-tuning and prompt engineering?mid
+
Fine-tuning updates model weights on task-specific data — requires compute but achieves top performance for narrow tasks. Prompt engineering crafts input text to steer a frozen model's output — zero-shot/few-shot, chain-of-thought, ReAct patterns. Prompt engineering is cheaper and more flexible; fine-tuning is better when you have consistent input distribution and need reproducible, task-specific behaviour.
Q09 Explain the KV-Cache and why it matters for inference.mid
+
During autoregressive generation, each new token must attend to all previous tokens. Without caching, you'd recompute K and V for all past tokens every step — O(n²) total FLOPs. The KV-Cache stores K and V tensors for all past positions and appends the new token's K, V at each step — reducing incremental generation to O(n) FLOPs. The trade-off is memory: a 7B model KV-cache at batch=1, seq=8K, FP16 costs ~2 GB.
Q10 What is temperature in text generation and how does it affect output?mid
+
Temperature T scales the logits before softmax: p_i = exp(logit_i / T). T=1 is standard sampling. T<1 sharpens the distribution (more deterministic/repetitive). T>1 flattens it (more random/creative). T→0 approaches greedy decoding. In practice: T=0.7 for creative tasks, T=0.1 for factual QA, T=1.0 with top-p=0.9 for conversational.
Advanced
Q11 Explain LoRA's mechanics and how rank r affects the trade-off.hard
+
LoRA factorises weight updates as ΔW = B·A where B∈ℝ^(d×r) and A∈ℝ^(r×k). B is zero-initialised so ΔW=0 at the start (same as base model). Only B and A are trained. Rank r controls expressiveness: low r (4–8) for simple domain adaptation; higher r (64–128) for complex tasks or larger distributional shifts. Too high r approaches full fine-tuning cost with less regularisation. Alpha/r ratio (commonly 2:1) controls the effective learning rate of the adapter.
Q12 How does Flash Attention achieve O(n) memory?hard
+
Standard attention materialises the full n×n attention matrix in HBM (GPU DRAM) — O(n²) memory. Flash Attention uses tiled computation: it processes small blocks of Q, K, V that fit in SRAM (on-chip, fast), computes local softmax with online normalisation (the log-sum-exp trick), and accumulates the output without ever writing the full attention matrix to HBM. The result: O(n) HBM reads/writes, ~4× less HBM I/O, and 2–4× wall-clock speedup for long sequences.
Q13 What are the differences between DPO, PPO-RLHF, and ORPO?hard
+
PPO-RLHF: Trains a separate reward model, then uses PPO to maximise it while KL-constraining against the reference policy. Complex, unstable, requires 4 models simultaneously.

DPO: Derives a closed-form equivalence to RLHF objective, training directly on (chosen, rejected) pairs with the reference model frozen. No separate reward model, more stable, but requires a good reference model.

ORPO: Odd Ratio Preference Optimisation (2024) combines SFT and alignment into a single loss function without needing a reference model — most compute-efficient of the three.
Q14 Explain Grouped Query Attention (GQA) and its motivation.hard
+
Multi-Head Attention (MHA) has H query, key, and value heads — KV-cache scales as O(n·H·d_head). Multi-Query Attention (MQA) uses 1 shared K,V for all query heads — 8× smaller KV-cache but lower quality. GQA is a middle ground: G groups of query heads share one K,V head per group (typically G=H/8). LLaMA-3 and Mistral use GQA — getting ~4–8× KV-cache reduction with near-MHA quality.
Q15 What is speculative decoding and how does it speed up LLM inference?hard
+
LLM inference is memory-bandwidth bound: generating 1 token uses the full model's parameters regardless. Speculative decoding uses a small "draft" model (e.g. 1B params) to generate K candidate tokens cheaply, then verifies all K tokens in one parallel forward pass through the large "target" model (e.g. 70B). Tokens that match the target's distribution are accepted; the first mismatch is corrected. If the draft quality is high, you get 2–4× wall-clock speedup because the large model processes K tokens in ~1 pass instead of K sequential passes.
Q16 What is catastrophic forgetting and how do you mitigate it when fine-tuning?hard
+
When fine-tuning on task-specific data, the model tends to overwrite pre-trained weights, losing general capabilities. Mitigations: (1) LoRA/PEFT — only update a small adapter, base weights unchanged; (2) Low learning rate with cosine decay; (3) Replay — mix in general data during fine-tuning; (4) EWC (Elastic Weight Consolidation) — penalise changes to weights important for previous tasks; (5) Early stopping based on held-out general benchmarks.
Q17 What causes hallucinations in LLMs and how can they be reduced?hard
+
Hallucinations arise from: (1) training on noisy/incorrect data; (2) the model maximising token probability rather than factual correctness; (3) overconfident decoding at high temperature; (4) insufficient factual grounding. Mitigations include: RAG (retrieval-augmented generation) to ground outputs in retrieved documents; calibration/uncertainty estimation; RLHF with factuality rewards; self-consistency sampling (generate multiple times, pick majority); tool use (calculators, search APIs); and lower temperature / beam search for factual tasks.
Q18 Describe the training objective of T5 and how it unifies NLP tasks.hard
+
T5 (Text-to-Text Transfer Transformer) frames every NLP task as a text-to-text problem: the input is a text string with a task prefix ("summarize:", "translate English to German:"), and the output is always generated text. Pre-training uses a "span corruption" objective — randomly mask contiguous spans and train the decoder to reconstruct them. This unified framing means one model and one training recipe handles classification, generation, translation, QA, and summarisation — just change the input prefix.
