Convolutional Neural Networks
Foundation · Deep Learning
CNNs apply learnable filters (kernels) across the spatial dimensions of an image, exploiting local connectivity and weight sharing to detect features hierarchically — edges → textures → parts → objects.
Weight Sharing Spatial Hierarchy Translation Invariant
Key layers: Conv2D → BatchNorm → ReLU → MaxPool
Output size: (W - K + 2P) / S + 1
Receptive field grows with depth
Use when: Image classification, feature extraction
Limitation: Lacks long-range dependency modeling
Python · PyTorch
import torch.nn as nn

class SimpleCNN(nn.Module):
    def __init__(self, num_classes=10):
        super().__init__()
        self.features = nn.Sequential(
            nn.Conv2d(3, 64, kernel_size=3, padding=1),
            nn.BatchNorm2d(64),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(2, 2),          # 112x112
            nn.Conv2d(64, 128, kernel_size=3, padding=1),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(2, 2),          # 56x56
        )
        self.classifier = nn.Linear(128 * 56 * 56, num_classes)

    def forward(self, x):
        x = self.features(x)
        x = x.view(x.size(0), -1)
        return self.classifier(x)
⛓️
ResNet — Residual Networks
Classification · 2015 · Microsoft Research
Introduces skip connections (residual shortcuts) that allow gradients to flow directly through the network, enabling training of very deep models (50–152+ layers) without degradation.
Skip Connections Deep Networks Vanishing Gradient Fix
Core idea: F(x) + x — learn residual, not mapping
Bottleneck block: 1×1 → 3×3 → 1×1 convolutions
Variants: ResNet-18/34/50/101/152
Use when: Backbone for classification & transfer learning
Top-1 ImageNet: ~76% (ResNet-50)
Python · PyTorch
from torchvision.models import resnet50
import torch.nn as nn

# Load pretrained ResNet-50
model = resnet50(pretrained=True)

# Fine-tune for custom task
num_features = model.fc.in_features
model.fc = nn.Linear(num_features, 5)  # 5 classes

# Freeze all except final layer
for param in model.parameters():
    param.requires_grad = False
model.fc.requires_grad_(True)

YOLO — You Only Look Once
Object Detection · Real-time
Reframes object detection as a single regression problem. Divides the image into a grid and directly predicts bounding boxes and class probabilities in one forward pass — enabling real-time detection.
Single-shot Real-time Anchor Boxes Grid-based
Output: S×S×(B×5 + C) tensor
Loss: localization + confidence + classification
NMS removes duplicate detections (IoU threshold)
YOLOv8 mAP: ~53.9 on COCO
Weakness: Struggles with small, clustered objects
Python · Ultralytics YOLOv8
from ultralytics import YOLO

# Load model
model = YOLO('yolov8n.pt')   # nano variant

# Inference
results = model('image.jpg')
for r in results:
    boxes = r.boxes.xyxy      # bounding boxes
    confs = r.boxes.conf      # confidence scores
    classes = r.boxes.cls     # class indices

# Fine-tune on custom data
model.train(data='data.yaml', epochs=50)

U-Net
Segmentation · Encoder-Decoder
Encoder-decoder architecture with skip connections between encoder and decoder levels. Originally designed for biomedical segmentation, now the go-to for any pixel-level prediction task.
Skip Connections Encoder-Decoder Pixel-level
Encoder: Contracting path (CNN + MaxPool)
Decoder: Expansive path (Upsample + Conv)
Skip connections preserve spatial information
Loss: Binary/multi-class cross-entropy + Dice loss
Best for: Medical imaging, satellite imagery
Python · segmentation-models-pytorch
import segmentation_models_pytorch as smp

model = smp.Unet(
    encoder_name="resnet34",
    encoder_weights="imagenet",
    in_channels=3,
    classes=1,        # binary segmentation
)

# Dice + BCE combined loss
loss_fn = smp.losses.DiceLoss(mode='binary')
# Output: (B, 1, H, W) probability mask

Vision Transformer (ViT)
Classification · Transformer-based · 2020
Applies the Transformer architecture directly to sequences of image patches. Splits image into fixed-size patches (16×16), linearly embeds them, adds position encodings, and processes with standard Transformer encoder.
Self-Attention Patch Embeddings Global Context Scalable
Input: Image → N patches → linear embed
Patches: 196 patches for 224×224 (16×16 patch)
Class token [CLS] aggregates global features
Advantage: Global receptive field from layer 1
Needs: Large datasets or strong augmentation
Python · Hugging Face Transformers
from transformers import ViTForImageClassification
from transformers import ViTImageProcessor

processor = ViTImageProcessor.from_pretrained(
    'google/vit-base-patch16-224'
)
model = ViTForImageClassification.from_pretrained(
    'google/vit-base-patch16-224'
)

inputs = processor(images=image, return_tensors="pt")
outputs = model(**inputs)
logits = outputs.logits

Generative Adversarial Networks
Generative Model · Goodfellow 2014
Two networks compete: a Generator creates fake images from noise, while a Discriminator distinguishes real from fake. This adversarial training produces photorealistic image synthesis.
Adversarial Training Generator Discriminator Latent Space
Generator: z ~ N(0,1) → fake image
Discriminator: image → P(real)
Loss: min_G max_D [log D(x) + log(1 - D(G(z)))]
Mode collapse is main training challenge
Variants: DCGAN, StyleGAN2, CycleGAN, Pix2Pix
Python · PyTorch GAN Loss
# Discriminator step
real_loss = criterion(D(real_imgs), real_labels)
fake_loss = criterion(D(G(z).detach()), fake_labels)
d_loss = (real_loss + fake_loss) / 2

# Generator step
g_loss = criterion(D(G(z)), real_labels)
# G wants D to think its output is real

Faster R-CNN
Object Detection · Two-stage · 2015
Two-stage detector that introduces the Region Proposal Network (RPN) — a small CNN that proposes candidate object regions, replacing slow selective search with a fast trainable network.
Two-Stage RPN RoI Pooling High Accuracy
Stage 1 — RPN: Proposes ~2000 candidate regions
Stage 2 — Fast R-CNN head: Classifies each RoI
RoI Pooling aligns features to fixed size
mAP on COCO: ~37 (baseline ResNet-50)
Trade-off: Slower than YOLO but more accurate
Python · Torchvision
from torchvision.models.detection import fasterrcnn_resnet50_fpn

model = fasterrcnn_resnet50_fpn(pretrained=True)
model.eval()

# Inference
with torch.no_grad():
    predictions = model([image_tensor])

boxes = predictions[0]['boxes']    # [N,4]
labels = predictions[0]['labels']  # [N]
scores = predictions[0]['scores']  # [N]

SIFT — Scale Invariant Feature Transform
Classical Feature Detection · Lowe 2004
Detects and describes local features in images that are invariant to scale, rotation, and partially to illumination changes. Gold standard for image matching and stitching before deep learning.
Scale Invariant Rotation Invariant 128-dim Descriptor DoG
Steps: Scale-space extrema → keypoint refinement → orientation assignment → descriptor
DoG (Difference of Gaussians) detects keypoints
Descriptor: 4×4 spatial bins × 8 orientation bins = 128D
Matching: Nearest neighbor with Lowe's ratio test
Use case: Panorama stitching, image retrieval, SLAM
Python · OpenCV
import cv2

sift = cv2.SIFT_create()
img = cv2.imread('image.jpg', 0)  # grayscale
keypoints, descriptors = sift.detectAndCompute(img, None)

# Match features between two images

Mask R-CNN
Instance Segmentation · He et al. 2017
Extends Faster R-CNN by adding a parallel branch that predicts binary segmentation masks for each detected object. Uses RoIAlign (bilinear interpolation) instead of RoI Pooling for precise spatial alignment.
Instance Segmentation RoIAlign Multi-task Loss
Heads: Classification + BBox regression + Mask
Mask head: Small FCN on each RoI → 28×28 binary mask
RoIAlign preserves exact spatial locations
Loss: L_cls + L_box + L_mask (binary cross-entropy)
Extensions: Keypoint detection, pose estimation
🌊
Optical Flow
Video Analysis · Motion Estimation
Estimates the apparent motion of objects between consecutive frames. Dense optical flow (Farneback) provides motion vectors for every pixel; sparse methods (Lucas-Kanade) track specific keypoints.
Motion Estimation Temporal Brightness Constancy
Assumption: I(x,y,t) = I(x+dx, y+dy, t+dt)
Sparse (LK): Pyramid-based tracking of keypoints
Dense (Farneback): Flow for every pixel using polynomials
Deep methods: FlowNet, PWC-Net, RAFT
Use case: Object tracking, action recognition, video stabilization
Python · OpenCV Dense Flow
import cv2
import numpy as np

prev_gray = cv2.cvtColor(prev_frame, cv2.COLOR_BGR2GRAY)
curr_gray = cv2.cvtColor(curr_frame, cv2.COLOR_BGR2GRAY)

flow = cv2.calcOpticalFlowFarneback(
    prev_gray, curr_gray, None,
    pyr_scale=0.5, levels=3, winsize=15,
    iterations=3, poly_n=5, poly_sigma=1.2, flags=0
)
magnitude, angle = cv2.cartToPolar(flow[...,0], flow[...,1])

INTERVIEW Q&A

50 Questions
Basic What is computer vision and how does it differ from image processing?
▼
Image Processing focuses on transforming or enhancing images using mathematical operations (blurring, sharpening, edge detection) — the output is still an image. It's signal processing applied to images.

Computer Vision is a broader AI field that aims to extract semantic meaning from images — the output is an interpretation (e.g., "there is a cat in the top-left corner"). It uses image processing as a tool but adds machine learning and deep learning to understand visual content.

Analogy: Image processing is like adjusting your glasses; computer vision is like understanding what you're seeing.
Basic Explain the concept of a convolution in CNNs. What does a filter detect?
▼
A convolution slides a small learnable matrix (kernel/filter) across the input image, computing the dot product at each position. The result is a feature map that highlights where the filter's pattern is present.

What filters learn:
  • Early layers: Low-level features — edges (horizontal, vertical, diagonal), corners, blobs
  • Middle layers: Textures, patterns, simple shapes
  • Deep layers: High-level concepts — eyes, wheels, faces, objects
Key properties: Local connectivity (only looks at small region), weight sharing (same filter across all positions), translation equivariance (feature detected anywhere in image).
Basic What is the difference between padding='same' and padding='valid' in CNNs?
▼
padding='valid': No padding. Output shrinks by (K-1) in each dimension for a kernel of size K. A 5×5 conv on 32×32 → 28×28.

padding='same': Adds zeros around the input so output spatial dimensions match input (for stride=1). Preserves spatial information — crucial for segmentation networks where you need pixel-level output matching input size.

Formula: Output = (W - K + 2P) / S + 1, where P = (K-1)/2 for 'same' padding with stride 1.
Basic What is pooling and why is it used? Max Pooling vs Average Pooling?
▼
Pooling down-samples feature maps, reducing spatial dimensions. Benefits: reduces computation, introduces translation invariance, controls overfitting.

Max Pooling: Takes the maximum value in each window. Preserves the most prominent feature (best activation). Most commonly used — good at detecting whether a feature is present.

Average Pooling: Takes the mean. Smoother, preserves more spatial information. Used in Global Average Pooling (GAP) before the final classification head in modern networks.

Global Average Pooling (GAP) replaces fully connected layers, reducing parameters drastically — used in ResNet, EfficientNet, etc.
Basic What are the common color spaces used in CV? When would you use HSV over RGB?
▼
RGB: Red, Green, Blue channels. Natural for most deep learning models. Color and brightness are intertwined.

Grayscale: Single channel, luminance only. Used when color isn't informative (e.g., medical imaging, edge detection).

HSV (Hue, Saturation, Value): Separates color (hue) from brightness (value). Use when: color-based segmentation (e.g., "find all red objects"), because filtering on hue is robust to lighting changes. Classic example: skin detection, ball tracking.

LAB: Perceptually uniform. L = lightness, A/B = color opponents. Use for perceptual image comparison, color correction.

YCbCr: Used in image compression (JPEG), separates luma from chroma.
Intermediate Why do ResNets work so well? Explain the vanishing gradient problem and how skip connections fix it.
▼
Vanishing Gradient Problem: In deep networks, gradients propagated back through many layers become exponentially smaller (multiplied by weights <1 repeatedly). Early layers receive near-zero gradients and barely update — learning stalls.

ResNet's solution — Skip connections: The residual block computes output = F(x) + x. The identity shortcut x provides a gradient highway — gradients can flow directly to earlier layers without passing through all the non-linear transformations.

Why residuals work: It's easier to learn a small residual change F(x) ≈ 0 (identity mapping as baseline) than to learn a full complex transformation. Many layers can effectively become near-identity, and the network decides how many layers to actually "use."

Result: Stable training of 50, 101, 152+ layer networks.
Intermediate What is Batch Normalization? Why is it critical for training deep CNNs?
▼
Batch Normalization normalizes the activations of a layer across a mini-batch, then applies learnable scale (γ) and shift (β) parameters.

Why it helps:
  • Reduces Internal Covariate Shift — distribution of activations no longer changes wildly during training
  • Allows higher learning rates → faster convergence
  • Acts as a regularizer (slight noise from batch statistics) → less need for Dropout
  • Reduces sensitivity to weight initialization
At inference: Uses running statistics (mean/variance accumulated during training), not batch statistics.

Variants: Layer Norm (used in Transformers), Instance Norm (style transfer), Group Norm (small batch sizes, object detection).
Intermediate What is IoU? How is mAP computed for object detection?
▼
IoU (Intersection over Union): IoU = Area of Intersection / Area of Union. Measures overlap between predicted and ground-truth bounding box. IoU ≥ 0.5 is typically considered a correct detection.

mAP (mean Average Precision) computation:
  • For each class: rank predictions by confidence score
  • Compute Precision and Recall at each threshold
  • AP = area under the Precision-Recall curve
  • mAP = mean of AP across all classes
mAP@50: IoU threshold = 0.5
mAP@50:95: Average over IoU thresholds 0.5, 0.55, ..., 0.95 (harder, used in COCO benchmark)
Intermediate Explain Non-Maximum Suppression (NMS). What problem does it solve?
▼
Problem: Object detectors (especially one-stage models like YOLO) produce many overlapping bounding boxes for the same object, each with a confidence score.

NMS algorithm:
  • Sort all boxes by confidence score (descending)
  • Keep the highest-scoring box
  • Remove all other boxes whose IoU with this box exceeds a threshold (typically 0.5)
  • Repeat for remaining boxes
Soft-NMS: Instead of hard removal, reduce scores of overlapping boxes proportionally. Better for densely packed objects (crowds).

NMS-free detectors: DETR uses set-based prediction with Hungarian matching, eliminating the need for NMS entirely.
Intermediate What is the Feature Pyramid Network (FPN) and why is it important for detection?
▼
Problem: Objects exist at multiple scales. Small objects are best detected in early (high-resolution) layers; large objects in later (semantically rich) layers. No single scale works for all sizes.

FPN solution: Creates a top-down pathway that merges high-level semantics with high-resolution features via lateral connections. Produces multi-scale feature maps (P2, P3, P4, P5) each of which is both high-resolution AND semantically strong.

Impact: Dramatically improves small object detection. Now used in virtually all modern detectors: Faster R-CNN+FPN, RetinaNet, YOLOv5/8, DETR.
Intermediate What is the difference between semantic, instance, and panoptic segmentation?
▼
Semantic Segmentation: Assigns a class label to every pixel. Does NOT distinguish between different instances of the same class. All cars are labeled "car." Models: FCN, DeepLab, SegFormer.

Instance Segmentation: Detects each object as a separate instance AND provides a pixel mask for each. Car #1 and Car #2 are distinct. Models: Mask R-CNN, SOLOv2.

Panoptic Segmentation: Unifies both — assigns each pixel both a semantic class AND an instance ID. "Things" (countable objects) get instance IDs; "stuff" (uncountable regions like sky, grass) gets only semantic labels. Models: Panoptic FPN, Mask2Former.

Difficulty: Semantic < Instance < Panoptic
Intermediate What is transfer learning in CV? What layers should you freeze vs fine-tune?
▼
Transfer learning uses weights pre-trained on large datasets (ImageNet) as initialization for a new task, rather than training from scratch.

Why it works: Early CNN layers learn universal features (edges, textures) that transfer across tasks. Only later layers are task-specific.

Strategy:
  • Small similar dataset: Freeze all conv layers, train only classifier head
  • Large similar dataset: Fine-tune all layers with small learning rate
  • Small different dataset: Freeze early layers, fine-tune later layers + head
  • Large different dataset: Fine-tune entire network (but start from pretrained weights)
Tip: Always use a 10× smaller learning rate for pretrained layers vs new layers.
Intermediate How would you handle class imbalance in a CV dataset (e.g., 1000 cats, 50 dogs)?
▼
Multiple strategies:

Data-level:
  • Oversample minority class (repeat/augment dog images)
  • Undersample majority class (remove some cat images)
  • SMOTE-like synthesis (generate new minority samples)
Algorithm-level:
  • Class-weighted loss: assign higher loss weight to minority class weight = N_total / (N_classes × N_class_i)
  • Focal Loss: down-weights easy/frequent examples, focuses on hard/rare ones
Evaluation: Never use accuracy on imbalanced data — use F1, precision/recall, AUC-ROC, or confusion matrix.
Advanced How does the Vision Transformer (ViT) differ from CNNs, and what are its trade-offs?
▼
Key differences:

CNNs: Inductive biases — locality (only local connections), translation equivariance. Good with limited data. Receptive field grows gradually.

ViT: No inductive bias — global self-attention from layer 1 allows every patch to attend to every other patch. Can learn arbitrary relationships. Requires much more data (JFT-300M, ImageNet-21k) or strong regularization.

ViT process: Split into 16×16 patches → flatten + linear projection → add position embedding → Transformer encoder blocks → classify [CLS] token

ViT advantages: Better at capturing long-range dependencies, scales better with data/compute, transferable representations (DINO, MAE)

ViT weaknesses: Data hungry, quadratic attention complexity O(N²), less efficient on small datasets without pretraining.
Advanced Explain Depthwise Separable Convolutions. Why do MobileNets use them?
▼
Standard Conv: K×K×C_in×C_out operations = expensive

Depthwise Separable Conv splits into two steps:
  • Depthwise conv: Apply one K×K filter per input channel independently (K×K×C_in)
  • Pointwise conv: 1×1 conv mixes channel information (1×1×C_in×C_out)
Computation reduction: 1/C_out + 1/K² ≈ 8–9× fewer operations for 3×3 conv

Why MobileNet uses it: Target is edge deployment (mobile phones, IoT). Achieves 70–80% of the accuracy of full models at <10× the compute cost.

Also used in: Xception, EfficientNet (inverted residuals with depthwise sep convs).
Advanced How does DETR (Detection Transformer) work and why is it novel?
▼
DETR novelty: First end-to-end object detector with no hand-designed components (no anchor boxes, no NMS, no region proposal networks).

Architecture:
  • CNN backbone extracts features
  • Transformer encoder processes feature map
  • Transformer decoder takes N learned object queries and attends to encoder output
  • Each query predicts one object (class + box)
Training: Hungarian algorithm finds optimal bipartite matching between N predictions and M ground truth objects. Unmatched predictions → "no object" class.

Limitations: Slow convergence (500 epochs), poor small object detection, high computational cost for high-res images.

Follow-ups: Deformable DETR, DAB-DETR, RT-DETR (faster, better small objects).
Advanced What is the Dice Loss? When is it better than Cross-Entropy for segmentation?
▼
Dice Loss: 1 - (2 × |P ∩ G|) / (|P| + |G|), where P = prediction, G = ground truth

Equivalent to 1 - Dice Coefficient (F1 score at pixel level)

When Dice Loss is better:
  • Class imbalance: Background pixels vastly outnumber foreground. Cross-entropy can achieve low loss by predicting all-background. Dice Loss normalizes by total foreground + predicted foreground
  • Medical image segmentation (small tumors vs large background)
  • Satellite imagery (roads, vessels)
Common practice: Use BCE + Dice combined loss — BCE provides stable gradients early in training; Dice directly optimizes the metric you care about.
Advanced How would you deploy a real-time object detection model on edge hardware?
▼
Pipeline:

1. Model Selection: Choose lightweight architecture (YOLOv8n, MobileNet-SSD, NanoDet)

2. Optimization:
  • Quantization: FP32 → INT8 (4× smaller, ~3× faster with minimal accuracy loss)
  • Pruning: Remove unimportant weights/channels
  • Knowledge Distillation: Train small model to mimic large model
3. Export: PyTorch → ONNX → TensorRT (NVIDIA) or TFLite (mobile) or OpenVINO (Intel)

4. Hardware acceleration: NVIDIA Jetson (TensorRT), Coral TPU (TFLite), Hailo-8, or Apple Neural Engine

5. Batching strategy: For video, process every 2nd-3rd frame; use tracking between detections

Target: <20ms latency for 50 FPS real-time processing
Advanced Explain the role of attention mechanisms in modern CV. Self-attention vs cross-attention?
▼
Attention in CV: Allows the model to focus on relevant spatial regions when making predictions, rather than treating all locations equally.

Self-Attention: Each position attends to all other positions in the same feature map. Used in ViT encoder — a patch of the "eye" can directly attend to the patch of the "ear" without going through intermediate layers. Captures global context instantly.

Cross-Attention: Queries from one source attend to keys/values from another. Used in DETR decoder — object queries attend to encoder's feature map to localize objects. Used in CLIP — text tokens attend to image patches.

Channel Attention (SE-Net): Recalibrates channel-wise feature responses — learns "which channels matter."

Spatial Attention: Learns "where to look" in the spatial dimensions — used in CBAM module combined with channel attention.
Basic What is data augmentation and why is it essential in CV?
▼
Data augmentation artificially increases the training set size and diversity by applying random transformations to existing images while preserving their labels.

Why essential:
  • Prevents overfitting on small datasets
  • Teaches the model invariances (rotated car is still a car)
  • Improves generalization to real-world variations (lighting, viewpoint, occlusion)
  • Cheap alternative to collecting more data
Types: Geometric (flip, rotate, crop, perspective), Photometric (brightness, contrast, noise, blur), Advanced (Mixup, CutMix, Mosaic)

Key library: Albumentations — faster and more flexible than torchvision transforms
Intermediate What is the receptive field of a CNN? How do you calculate and increase it?
▼
Receptive field: The region of the input image that influences a particular output neuron. Larger receptive field = neuron "sees" more context.

Growth: With stride=1 and K×K kernels, each layer adds (K-1) to receptive field. Two 3×3 layers = 5×5 effective RF. Three = 7×7 effective RF (but with fewer parameters than a single 7×7 conv).

Ways to increase RF:
  • Dilated/Atrous convolutions: Insert gaps in the kernel — 3×3 with dilation=2 has same RF as 5×5 but fewer parameters. Used in DeepLab.
  • Max pooling / strided convolutions: Reduce spatial size, increasing RF for subsequent layers
  • Deeper networks: More layers = larger RF
Why it matters: For segmentation, neurons in the output need a large RF to have context about the entire image region.
Advanced What is Contrastive Learning and how does CLIP work for vision-language alignment?
▼
Contrastive Learning: Learn representations by pulling together embeddings of similar pairs and pushing apart embeddings of dissimilar pairs. No labels needed — uses data augmentations or natural pairings as supervision signal.

SimCLR/MoCo: Two augmented views of same image should have similar embeddings; views from different images should be dissimilar. Loss: InfoNCE (normalized cross-entropy over negative pairs).

CLIP (OpenAI):
  • Trains image encoder and text encoder jointly on 400M image-text pairs from the web
  • Matches image embeddings with their correct text description (contrastive loss over batch)
  • At inference: zero-shot classification by computing similarity between image embedding and text embeddings of all class names
Power: CLIP's embeddings generalize remarkably — can classify objects never seen during training by describing them in text. Foundation for DALL-E, Stable Diffusion (CLIP text encoder), SAM.
Intermediate What are anchor boxes? Why did modern detectors (YOLOv8, DETR) move away from them?
▼
Anchor boxes: Pre-defined bounding boxes of various scales and aspect ratios placed at each grid cell. The detector predicts offsets from these anchors. Examples: YOLOv3 uses 9 anchors (3 per FPN level), Faster R-CNN uses 9 anchors per location (3 scales × 3 ratios).

Problems with anchors:
  • Hyperparameters that must be tuned per dataset (k-means clustering on ground truth boxes)
  • Miss objects with unusual aspect ratios
  • Imbalance between foreground and background anchors
  • Complicated multi-scale matching logic
Anchor-free approaches:
  • YOLOv8: Predicts bounding box as distance from object center to 4 sides
  • DETR: Uses learned object queries — no predefined spatial priors
  • FCOS: Center-ness branch to suppress off-center predictions
Advanced What is Neural Architecture Search (NAS)? How did EfficientNet use compound scaling?
▼
NAS automates the design of neural network architectures using search algorithms (reinforcement learning, evolutionary algorithms, differentiable NAS). The search space defines possible operations; the algorithm finds the best combination for a given compute budget.

EfficientNet (Tan & Le, 2019): Instead of arbitrary scaling, found that you should scale width, depth, and resolution together — compound scaling.

Compound coefficient φ:
  • Depth: d = 1.2^φ
  • Width: w = 1.1^φ
  • Resolution: r = 1.15^φ
These coefficients are found by NAS on a baseline (EfficientNet-B0). EfficientNet-B7 achieves 84.3% ImageNet with 66M params vs ResNet-152 (77.8%, 60M params).

Key insight: All three dimensions contribute — scaling only depth or width yields diminishing returns.
Basic What are the standard ImageNet preprocessing steps? Why normalize with ImageNet mean/std?
▼
Standard preprocessing:
  • Resize image to 256×256 (or 232 for EfficientNet)
  • Random crop to 224×224 during training; center crop for evaluation
  • Normalize with ImageNet statistics
ImageNet mean/std:
  • mean = [0.485, 0.456, 0.406] (per channel R, G, B)
  • std = [0.229, 0.224, 0.225]
Why normalize: Pretrained models saw training data normalized this way. Using these exact values ensures activations operate in the range the weights expect, enabling effective transfer learning. Without this, fine-tuning fails or converges poorly.

For models trained from scratch, compute mean/std of your own dataset.
Intermediate What is Grad-CAM and how does it help explain CV models?
▼
Grad-CAM (Gradient-weighted Class Activation Mapping): Produces heatmaps showing which image regions the model focused on when making a prediction — without modifying the architecture.

How it works:
  • Forward pass → compute class score for target class
  • Backpropagate gradient to last convolutional layer
  • Compute neuron importance weights: global average pool the gradients
  • Weighted sum of forward activations → heatmap
  • Apply ReLU (keep positive activations) → upscale to input size
Uses: Debug model failures ("it predicted cat because of the grass"), regulatory compliance, medical AI explanation, bias detection ("the model looked at race instead of the tumor")
Advanced Explain the difference between RoI Pooling and RoIAlign. Why does it matter for segmentation?
▼
RoI Pooling (Faster R-CNN): Quantizes (rounds) the floating-point RoI coordinates to integer pixel boundaries, then applies max pooling within bins. The double quantization introduces misalignment errors — up to half a cell width off.

RoIAlign (Mask R-CNN): Avoids quantization entirely. Uses bilinear interpolation to compute feature values at exact floating-point locations within each bin. Much more spatially precise.

Why it matters for segmentation: Predicting pixel-accurate masks requires exact spatial correspondence. In Mask R-CNN, even 1-2 pixel misalignment between the RoI features and the mask output significantly hurts quality. RoIAlign improved Mask AP by ~3 points over RoI Pooling.

For bounding box detection the quantization error is acceptable; for masks it is not.
Basic What is OpenCV and what are its most useful functions for CV engineers?
▼
OpenCV (Open Source Computer Vision Library) is the most widely used CV library — optimized C++ with Python bindings.

Essential functions:
  • cv2.imread / imwrite / imshow — I/O basics
  • cv2.resize, warpAffine, warpPerspective — geometric transforms
  • cv2.cvtColor — color space conversions
  • cv2.GaussianBlur, medianBlur, bilateralFilter — smoothing
  • cv2.Canny — edge detection
  • cv2.findContours, drawContours — contour analysis
  • cv2.threshold, adaptiveThreshold — segmentation
  • cv2.VideoCapture — video I/O
  • cv2.dnn.readNet — run ONNX/Caffe/TF models
