TOPIC 13

Heap

// BINARY HEAP · MIN-HEAP · MAX-HEAP · PRIORITY QUEUE · HEAP SORT

📐
A heap is a complete binary tree stored as an array where parent[i] = arr[(i-1)//2], left_child = arr[2i+1], right_child = arr[2i+2]. Min-heap: parent ≤ children. Max-heap: parent ≥ children. Python's heapq is a min-heap only.

Operations & Complexity

  • insert / heappush → O(log n)
  • extract-min/max / heappop → O(log n)
  • peek min/max → O(1)
  • build heap from array → O(n)
  • heapify (sift down) → O(log n)
  • delete arbitrary element → O(log n)

Key Applications

  • Dijkstra's shortest path
  • Prim's MST algorithm
  • Kth largest / smallest element
  • Merge K sorted lists
  • Median of data stream
  • Task / job scheduling

Python heapq Tips

  • Default: min-heap only
  • Max-heap: negate values (-x)
  • Tuples: heap by first element
  • heapq.nlargest(k, arr) → O(n log k)
  • heapq.nsmallest(k, arr) → O(n log k)
  • heapq.heapify(arr) → O(n) in-place

import heapq

# === MEDIAN OF DATA STREAM ===
# Two heaps: max-heap (left half) + min-heap (right half)
class MedianFinder:
    def __init__(self):
        self.lo = []   # max-heap (negate values)
        self.hi = []   # min-heap

    def add_num(self, num):
        heapq.heappush(self.lo, -num)
        # balance: max of lo must be ≤ min of hi
        heapq.heappush(self.hi, -heapq.heappop(self.lo))
        if len(self.lo) < len(self.hi):
            heapq.heappush(self.lo, -heapq.heappop(self.hi))

    def find_median(self):
        if len(self.lo) > len(self.hi): return -self.lo[0]
        return (-self.lo[0] + self.hi[0]) / 2

# === MERGE K SORTED LISTS ===
def merge_k_lists(lists):
    heap = []
    for i, node in enumerate(lists):
        if node: heapq.heappush(heap, (node.val, i, node))

# === TOP K FREQUENT ELEMENTS ===
def top_k_frequent(nums, k):
    from collections import Counter
    return [x for x, _ in Counter(nums).most_common(k)]

# === HEAP SORT ===
def heap_sort(arr):
    heapq.heapify(arr)          # O(n)
    return [heapq.heappop(arr) for _ in range(len(arr))]  # O(n log n)
TOPIC 14

Graphs

// BFS · DFS · DIJKSTRA · BELLMAN-FORD · TOPOLOGICAL SORT · MST · SCC

🔵 BFS — Breadth-First Search

Queue-based. Explores level by level. Finds shortest path in unweighted graphs. O(V+E). Used for: shortest path, connected components, bipartite check.

🔴 DFS — Depth-First Search

Stack/recursion-based. Explores as deep as possible. O(V+E). Used for: cycle detection, topological sort, SCCs, path finding, flood fill.

🟡 Dijkstra's Algorithm

Single-source shortest path for non-negative weights. Min-heap + greedy. O((V+E) log V). Does NOT work with negative edges.

🟠 Bellman-Ford

Single-source shortest path with negative weights. Relax all edges V-1 times. O(VE). Detects negative cycles. Slower but more general than Dijkstra.

🟢 Floyd-Warshall

All-pairs shortest path. O(V³). Simple DP: dp[i][j] = min path from i to j. Works with negative edges (not cycles). Best for dense graphs.

🟣 Topological Sort

Linear ordering of DAG nodes. Kahn's (BFS + in-degree) or DFS post-order. O(V+E). Used for: build systems, dependency resolution, course scheduling.

🔵 Kruskal's MST

Sort edges by weight, add if no cycle (Union-Find). O(E log E). Greedy. Works well for sparse graphs. Minimum Spanning Tree.

🔴 Prim's MST

Grow MST from a starting node. Use min-heap to pick cheapest edge. O((V+E) log V). Better for dense graphs than Kruskal's.

🟡 Union-Find (DSU)

Disjoint Set Union. Tracks connected components. find() with path compression: ~O(1). Used with Kruskal's, detecting cycles, dynamic connectivity.

graph_algorithms.py 
from collections import deque, defaultdict
import heapq

# === DIJKSTRA'S ALGORITHM ===
def dijkstra(graph, src, n):
    # graph[u] = [(dist, v), ...]
    dist = [float('inf')] * n; dist[src] = 0
    heap = [(0, src)]
    while heap:
        d, u = heapq.heappop(heap)
        if d > dist[u]: continue   # stale entry
        for w, v in graph[u]:
            if dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                heapq.heappush(heap, (dist[v], v))
    return dist

# === UNION-FIND with path compression + union by rank ===
class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n

    def find(self, x):
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])  # path compression
        return self.parent[x]

    def union(self, x, y):
        px, py = self.find(x), self.find(y)
        if px == py: return False
        if self.rank[px] < self.rank[py]: px, py = py, px
        self.parent[py] = px
        if self.rank[px] == self.rank[py]: self.rank[px] += 1
        return True

# === TOPOLOGICAL SORT — Kahn's BFS ===
def topo_sort(n, edges):
    adj = defaultdict(list); indeg = [0] * n
    for u, v in edges: adj[u].append(v); indeg[v] += 1
    q = deque([i for i in range(n) if indeg[i] == 0])
    order = []
    while q:
        u = q.popleft(); order.append(u)
        for v in adj[u]:
            indeg[v] -= 1
            if indeg[v] == 0: q.append(v)
    return order if len(order) == n else []  # [] = cycle exists
TOPIC 15

Dynamic Programming

// MEMOIZATION · TABULATION · OVERLAPPING SUBPROBLEMS · OPTIMAL SUBSTRUCTURE

🔑
DP applies when: (1) Optimal substructure — optimal solution built from optimal sub-solutions, (2) Overlapping subproblems — same subproblems solved multiple times. If both hold, use DP to avoid redundant work.
01

1D DP — Linear Problems

State depends on a single variable. dp[i] = best result using first i elements.
Examples: Fibonacci, Climbing Stairs, House Robber, Coin Change, Jump Game, Decode Ways

02

2D DP — Grid / String Problems

State = dp[i][j] depends on two variables (position, length, indices).
Examples: LCS, Edit Distance, Unique Paths, Min Path Sum, Longest Palindromic Subsequence

03

Knapsack Variants

0/1 Knapsack, Unbounded Knapsack, Subset Sum, Partition Equal Subset, Target Sum, Combination Sum IV.
Pattern: choose or skip each item, optimize a capacity/target constraint.

04

Interval DP

dp[i][j] = optimal for subarray/substring from i to j. Process by length.
Examples: Matrix Chain Multiplication, Burst Balloons, Palindrome Partitioning

05

Bitmask DP

State includes a bitmask representing subset of elements used. O(2ⁿ × n).
Examples: TSP (Travelling Salesman), Minimum Cost to Visit All Nodes, Assignment Problem

06

DP on Trees / Graphs

Compute DP values for each node based on children. Root the tree, DFS, combine subtree results.
Examples: Diameter, Max Path Sum, House Robber III, Subtree sizes for DSU on Tree

dynamic_programming.py 
# === COIN CHANGE — Bottom-Up DP ===
def coin_change(coins, amount):
    dp = [float('inf')] * (amount + 1); dp[0] = 0
    for coin in coins:
        for a in range(coin, amount+1):
            dp[a] = min(dp[a], dp[a-coin] + 1)
    return dp[amount] if dp[amount] != float('inf') else -1

# === LONGEST COMMON SUBSEQUENCE ===
def lcs(s1, s2):
    m, n = len(s1), len(s2)
    dp = [[0]*(n+1) for _ in range(m+1)]
    for i in range(1, m+1):
        for j in range(1, n+1):
            if s1[i-1] == s2[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1
            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])
    return dp[m][n]

# === 0/1 KNAPSACK ===
def knapsack(weights, values, W):
    n = len(weights)
    dp = [0] * (W+1)
    for i in range(n):
        for w in range(W, weights[i]-1, -1):  # reverse to avoid reuse
            dp[w] = max(dp[w], dp[w-weights[i]] + values[i])
    return dp[W]

# === EDIT DISTANCE (Levenshtein) ===
def edit_dist(s, t):
    m, n = len(s), len(t)
    dp = [[j if i==0 else i if j==0 else 0
             for j in range(n+1)] for i in range(m+1)]
    for i in range(1, m+1):
        for j in range(1, n+1):
            if s[i-1]==t[j-1]: dp[i][j] = dp[i-1][j-1]
            else: dp[i][j] = 1 + min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1])
    return dp[m][n]

Greedy Algorithms

// LOCAL OPTIMAL · GLOBAL OPTIMAL · EXCHANGE ARGUMENT · PROOF

⚡
A greedy algorithm makes the locally optimal choice at each step. Works when: (1) Greedy choice property — local optimum leads to global optimum, (2) Optimal substructure. Proving correctness usually requires an exchange argument — show any deviation from greedy leads to equal or worse result.

Activity Selection

Sort by finish time. Always pick the next activity that starts after current ends. O(n log n). Gives maximum non-overlapping activities.

Huffman Coding

Build optimal prefix-free codes. Use min-heap: repeatedly merge two smallest frequencies. O(n log n). Foundation of lossless compression.

Fractional Knapsack

Sort by value/weight ratio. Take as much of highest ratio items as possible. O(n log n). Unlike 0/1 knapsack, greedy works here.

Jump Game

Track max reachable position. If current index exceeds max reach, return false. O(n) greedy beats O(n²) DP. Classic monotonic greedy.

Gas Station Circuit

If total gas ≥ total cost, solution exists. Start from last failure point. O(n). Classic greedy with circular array and running sum.

Minimum Platforms

Sort arrivals and departures separately. Use two pointers. At each point, count overlapping trains. O(n log n). Interval scheduling technique.

# === INTERVAL SCHEDULING — Max non-overlapping intervals ===
def erase_overlap_intervals(intervals):
    intervals.sort(key=lambda x: x[1])  # sort by end time
    end = float('-inf'); removed = 0
    for s, e in intervals:
        if s >= end: end = e   # no overlap, take it
        else: removed += 1       # overlap, remove it
    return removed

# === JUMP GAME II — Minimum jumps ===
def jump(nums):
    jumps = farthest = end = 0
    for i in range(len(nums)-1):
        farthest = max(farthest, i+nums[i])
        if i == end:
            jumps += 1; end = farthest
    return jumps

# === HUFFMAN ENCODING — Build optimal prefix tree ===
import heapq
def huffman(freq):
    heap = [[f, [sym, ""]] for sym, f in freq.items()]
    heapq.heapify(heap)
    while len(heap) > 1:
        lo = heapq.heappop(heap); hi = heapq.heappop(heap)
        for pair in lo[1:]: pair[1] = '0' + pair[1]
        for pair in hi[1:]: pair[1] = '1' + pair[1]
        heapq.heappush(heap, [lo[0]+hi[0]] + lo[1:] + hi[1:])
    return sorted(heap[0][1:], key=lambda p: len(p[-1]))

Backtracking

// EXPLORE · CHOOSE · UNCHOOSE · CONSTRAINT PRUNING · DECISION TREE

🌳
Backtracking = DFS on a decision tree. At each node: Choose an option, Explore further, Unchoose (backtrack). The power comes from pruning — cutting branches that can never lead to a valid solution early.

Permutations

Generate all n! arrangements. At each level, try each unused element. O(n × n!). Use visited boolean array or swap-based approach.

Combinations / Subsets

2ⁿ subsets. Choose to include or exclude each element. O(2ⁿ). For combinations(n,k): O(k × C(n,k)). Sort + start index avoids duplicates.

N-Queens

Place N queens on N×N board with no conflicts. Track columns and diagonals with sets. O(N!). Classic constraint satisfaction problem.

Sudoku Solver

Fill 9×9 grid satisfying row/col/box constraints. Try digits 1-9, backtrack on conflict. With pruning, solves most Sudokus quickly.

Word Search

Find word in grid by DFS from each cell. Mark visited, explore 4 directions, unmark on backtrack. O(M×N × 4^L) where L = word length.

Palindrome Partition

All ways to partition string into palindromes. At each cut point, check if prefix is palindrome, recurse on suffix. Prune non-palindrome prefixes.

# === GENERAL BACKTRACKING TEMPLATE ===
def backtrack(state, choices, result):

        result.append(list(state)); return
    for choice in choices:
        if is_valid(state, choice):
            state.append(choice)    # Choose

# === PERMUTATIONS — All orderings ===
def permute(nums):
    res = []
    def bt(path, used):
        if len(path) == len(nums): res.append(list(path)); return
        for i, n in enumerate(nums):
            if not used[i]:
                used[i] = True; path.append(n)
                bt(path, used)
                used[i] = False; path.pop()
    bt([], [False]*len(nums)); return res

# === N-QUEENS ===
def solve_n_queens(n):
    res = []; cols = set(); diag1 = set(); diag2 = set()
    def bt(row, board):
        if row == n: res.append([''.join(r) for r in board]); return
        for col in range(n):
            if col in cols or (row-col) in diag1 or (row+col) in diag2: continue
            cols.add(col); diag1.add(row-col); diag2.add(row+col)
            board[row][col] = 'Q'
            bt(row+1, board)
            cols.discard(col); diag1.discard(row-col); diag2.discard(row+col)
            board[row][col] = '.'
    bt(0, [['.']*n for _ in range(n)]); return res
TOPIC 18

Advanced DSA

// SEGMENT TREE · BIT · SPARSE TABLE · FLOYD CYCLE · BIT MANIPULATION · MATH

Segment Tree

O(n) build O(log n) query O(log n) update

Range queries (sum/min/max) and updates. 4n array storage. Supports lazy propagation for range updates. Essential for competitive programming.

Fenwick Tree (BIT)

O(n) build O(log n) query O(log n) update

Prefix sums with point updates. Uses bit trick: i += i & (-i) to traverse. Simpler code than segment tree, but limited to invertible operations.

Sparse Table

O(n log n) build O(1) query

Immutable range minimum/maximum queries in O(1). Precompute 2^j jumps. Cannot handle updates. Ideal for static arrays, LCA queries.

Bit Manipulation

AND, OR, XOR, shifts. Key tricks: x & (x-1) clears lowest bit, x & (-x) isolates lowest bit, x ^ x = 0. Used in subset enumeration, power of two checks, counting set bits.

Mathematical Algorithms

Sieve of Eratosthenes O(n log log n), GCD/LCM (Euclidean), Fast Exponentiation O(log n), Modular Inverse (Fermat's little theorem), Chinese Remainder Theorem.

Floyd's Cycle Detection

Fast and slow pointers for cycle detection in O(n) time O(1) space. Also finds cycle start. Works on linked lists, arrays (find duplicate), number sequences.

# === SEGMENT TREE — Sum with range update ===
class SegTree:
    def __init__(self, arr):
        self.n = len(arr)
        self.tree = [0] * (4 * self.n)
        self.build(arr, 0, 0, self.n-1)

    def build(self, arr, node, lo, hi):
        if lo == hi: self.tree[node] = arr[lo]; return
        mid = (lo+hi)//2
        self.build(arr, 2*node+1, lo, mid)
        self.build(arr, 2*node+2, mid+1, hi)
        self.tree[node] = self.tree[2*node+1] + self.tree[2*node+2]

    def query(self, l, r, node=0, lo=0, hi=None):
        if hi is None: hi = self.n-1
        if l > hi or r < lo: return 0
        if l <= lo and hi <= r: return self.tree[node]
        mid = (lo+hi)//2
        return (self.query(l, r, 2*node+1, lo, mid) +
                self.query(l, r, 2*node+2, mid+1, hi))

# === FENWICK TREE (BIT) ===
class BIT:
    def __init__(self, n): self.bit = [0]*(n+1); self.n = n
    def update(self, i, delta):
        while i <= self.n: self.bit[i] += delta; i += i & (-i)
    def prefix_sum(self, i):
        s = 0
        while i > 0: s += self.bit[i]; i -= i & (-i)
        return s

# === BIT MANIPULATION TRICKS ===
def bit_tricks(x):
    is_power_of_2 = x > 0 and (x & (x-1)) == 0
    lowest_set_bit = x & (-x)
    count_set_bits = bin(x).count('1')
    clear_lowest = x & (x-1)
    return is_power_of_2, lowest_set_bit, count_set_bits

# === SIEVE OF ERATOSTHENES ===
def sieve(n):
    is_prime = [True] * (n+1); is_prime[0] = is_prime[1] = False
    for i in range(2, int(n**.5)+1):
        if is_prime[i]:
            for j in range(i*i, n+1, i): is_prime[j] = False
    return [i for i, p in enumerate(is_prime) if p]

# === FAST EXPONENTIATION — O(log n) ===
def fast_pow(base, exp, mod):
    result = 1; base %= mod
    while exp > 0:
        if exp % 2 == 1: result = result * base % mod
        base = base * base % mod; exp //= 2
    return result
