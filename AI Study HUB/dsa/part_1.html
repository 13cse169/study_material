Essential Python Tricks for DSA

  • Multiple assignment: a, b = b, a — swap without temp variable
  • Slicing: arr[::−1] reverses a list in O(n); arr[l:r] creates subarray
  • bisect module: bisect_left/right for binary search on sorted lists in O(log n)
  • sys.setrecursionlimit(10**6): increase default recursion depth for deep DFS/recursion
  • Ternary: val = x if condition else y — clean inline conditional
  • enumerate(arr, start=1): index + value in loops — avoids separate counter
  • zip(*matrix): transpose a 2D matrix elegantly in one line

complexity_examples.py 
# O(1) — Constant
def get_first(arr): return arr[0]

# O(log n) — Binary search halves search space each step
def binary_search(arr, target):
    l, r = 0, len(arr) - 1
    while l <= r:
        m = (l + r) // 2
        if arr[m] == target: return m
        elif arr[m] < target: l = m + 1
        else: r = m - 1
    return -1

# O(n) — Single loop
def linear_scan(arr): return sum(arr)

# O(n²) — Nested loops
def bubble_sort(arr):
    for i in range(len(arr)):
        for j in range(len(arr)-i-1):
            if arr[j] > arr[j+1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]
📐

Space Complexity Rules

  • Only count extra space used (not input)
  • Recursive calls use O(depth) stack space
  • In-place algorithms use O(1) extra space
  • Memoization trades space for time
⚖️

Big-O Rules

  • Drop constants: O(2n) → O(n)
  • Drop lower terms: O(n² + n) → O(n²)
  • Sequential loops: add complexities
  • Nested loops: multiply complexities
🎯

Interview Target Complexities

  • n ≤ 20 → O(2ⁿ) or O(n!) acceptable
  • n ≤ 500 → O(n²) or O(n³)
  • n ≤ 10⁴ → O(n²) is fine
  • n ≤ 10⁶ → Need O(n log n) or O(n)
  • n ≤ 10⁸ → Need O(n) or O(log n)
TOPIC 03

Arrays

📊

Array Fundamentals

Contiguous memory, O(1) random access via index. Fixed-size in most languages; Python lists are dynamic arrays that double in size when full.

🪟

Sliding Window

Maintain a window of elements, expand/shrink from ends. Converts O(n²) brute force to O(n). Used for subarray sum, max substring, etc.

👉👈

Two Pointers

Left and right pointers moving toward each other (or same direction). Solves pair sum, palindrome check, container with most water in O(n).

➕

Prefix Sums

Pre-compute cumulative sums. Range sum query becomes O(1): sum(l,r) = prefix[r] - prefix[l-1]. Foundation for many DP problems.

🔀

Kadane's Algorithm

Maximum subarray sum in O(n). Track current sum and global max. Classic DP-on-arrays problem: if current becomes negative, reset to 0.

🗺️

2D Arrays / Matrices

Grid problems: BFS/DFS traversal, spiral order, rotate 90°, diagonal traversal. Matrix multiplication in O(n³). Sparse matrices use dicts.

array_patterns.py 
# === SLIDING WINDOW — Max sum subarray of size k ===
def max_sum_window(arr, k):
    window = sum(arr[:k])
    best = window
    for i in range(k, len(arr)):
        window += arr[i] - arr[i-k]
        best = max(best, window)
    return best

# === TWO POINTERS — Two sum in sorted array ===
def two_sum_sorted(arr, target):
    l, r = 0, len(arr) - 1
    while l < r:
        s = arr[l] + arr[r]
        if s == target: return [l, r]
        elif s < target: l += 1
        else: r -= 1

# === PREFIX SUM — Range sum query ===
def build_prefix(arr):
    prefix = [0] * (len(arr) + 1)
    for i, v in enumerate(arr):
        prefix[i+1] = prefix[i] + v
    return prefix
# range_sum(l, r) = prefix[r+1] - prefix[l]

# === KADANE'S ALGORITHM — Max subarray ===
def kadane(arr):
    cur = best = arr[0]
    for n in arr[1:]:
        cur = max(n, cur + n)
        best = max(best, cur)
    return best

TOPIC 04

Strings

🔤

String Basics in Python

Strings are immutable sequences. s[i] is O(1). Concatenation with + is O(n) — prefer ''.join(list). Slicing creates a new string in O(k).

🔎

Pattern Matching

Naive O(nm), KMP O(n+m), Rabin-Karp (rolling hash), Z-algorithm. Python's built-in str.find() uses a mix of Boyer-Moore and Horspool.

🔁

Palindromes

Expand around center: O(n²). Manacher's algorithm: O(n). Two-pointer from center. DP table for all palindromic substrings in O(n²).

#️⃣

Anagrams & Frequency

Counter/dict comparison: O(n). Sliding window with char frequency for substring problems. Sort-based anagram check: O(n log n).

string_algorithms.py 
# === KMP — Pattern Matching in O(n+m) ===
def kmp_search(text, pattern):
    def build_lps(p):
        lps = [0] * len(p); l = 0; i = 1
        while i < len(p):
            if p[i] == p[l]: l += 1; lps[i] = l; i += 1
            elif l: l = lps[l-1]
            else: i += 1
        return lps
    lps = build_lps(pattern)
    i = j = 0; results = []
    while i < len(text):
        if text[i] == pattern[j]: i += 1; j += 1
        if j == len(pattern):
            results.append(i-j); j = lps[j-1]
        elif i < len(text) and text[i] != pattern[j]:
            if j: j = lps[j-1]
            else: i += 1
    return results

# === Longest Palindromic Substring — Expand Around Center ===
def longest_palindrome(s):
    res = ""
    for i in range(len(s)):
        for l, r in [(i,i), (i,i+1)]:   # odd and even
            while l>=0 and r<len(s) and s[l]==s[r]:
                l-=1; r+=1
            if r-l-1 > len(res): res = s[l+1:r]
    return res

# === Sliding Window — Longest substring without repeats ===
def length_of_longest_substring(s):
    seen = {}; l = res = 0
    for r, c in enumerate(s):
        if c in seen: l = max(l, seen[c]+1)
        seen[c] = r
        res = max(res, r-l+1)
    return res

TOPIC 05

Recursion

🔄
Recursion is a function calling itself with a smaller subproblem. Every recursive solution needs: (1) Base case — when to stop, (2) Recursive case — smaller subproblem, (3) Progress — must move toward base case.
📞

Call Stack

Each call adds a frame to the stack. Stack overflow if depth is too large. Python default limit: 1000. Use sys.setrecursionlimit() for deeper recursion.

🌲

Recursion Tree

Visualize recursive calls as a tree. Count total nodes to find time complexity. Overlapping subproblems → use memoization (DP).

↩️

Tail Recursion

When the recursive call is the last operation. Python does NOT optimize tail calls (unlike Scheme/Scala). Convert to iteration for O(1) space.

📝

Memoization

Cache results of expensive function calls. @functools.lru_cache decorator in Python. Converts exponential to polynomial time for overlapping subproblems.

recursion_examples.py 
from functools import lru_cache

# === Fibonacci — without and with memoization ===
def fib_naive(n):           # O(2^n) — very slow
    if n <= 1: return n
    return fib_naive(n-1) + fib_naive(n-2)

@lru_cache(maxsize=None)
def fib_memo(n):           # O(n) — with caching
    if n <= 1: return n
    return fib_memo(n-1) + fib_memo(n-2)

# === Tower of Hanoi — Classic recursion ===
def hanoi(n, src, aux, dst):
    if n == 1:
        print(f"Move disk 1 from {src} to {dst}"); return
    hanoi(n-1, src, dst, aux)
    print(f"Move disk {n} from {src} to {dst}")
    hanoi(n-1, aux, src, dst)

# === Merge Sort — Divide & Conquer recursion ===
def merge_sort(arr):
    if len(arr) <= 1: return arr
    mid = len(arr) // 2
    L = merge_sort(arr[:mid])
    R = merge_sort(arr[mid:])
    return merge(L, R)

def merge(L, R):
    res = []; i = j = 0
    while i < len(L) and j < len(R):
        if L[i] <= R[j]: res.append(L[i]); i+=1
        else: res.append(R[j]); j+=1
    return res + L[i:] + R[j:]
⚠️
Common Recursion Mistakes: Forgetting base case (infinite recursion), modifying shared state without care, not handling n=0 or empty input, ignoring stack overflow risk for large n.
TOPIC 06

Searching Algorithms

🔍

Linear Search

Scan every element. O(n) time, O(1) space. Works on unsorted data. Best for small arrays or when data is unordered.

🎯

Binary Search

Requires sorted input. Halves search space each step → O(log n). Template: find exact value, find leftmost, find rightmost insertion point.

📐

Binary Search on Answer

When the answer is monotonic (e.g., "find min k satisfying condition"), binary search on the answer space. Powerful technique for optimization problems.

🌊

Ternary Search

For unimodal functions — find maximum/minimum. Divides into thirds, eliminates one-third each iteration. O(log₃ n) comparisons.

searching_algorithms.py 
# === BINARY SEARCH — Standard Template ===
def binary_search(arr, target):
    l, r = 0, len(arr) - 1
    while l <= r:
        mid = l + (r - l) // 2  # avoids integer overflow
        if arr[mid] == target: return mid
        elif arr[mid] < target: l = mid + 1
        else: r = mid - 1
    return -1

# === LEFTMOST INSERTION — bisect_left equivalent ===
def lower_bound(arr, target):
    l, r = 0, len(arr)
    while l < r:
        mid = (l + r) // 2
        if arr[mid] < target: l = mid + 1
        else: r = mid
    return l

# === BINARY SEARCH ON ANSWER — Min capacity to ship in D days ===
def ship_within_days(weights, D):
    def can_ship(cap):
        days = 1; cur = 0
        for w in weights:
            if cur + w > cap: days += 1; cur = 0
            cur += w
        return days <= D
    l, r = max(weights), sum(weights)
    while l < r:
        mid = (l + r) // 2
        if can_ship(mid): r = mid
        else: l = mid + 1
    return l

# === SEARCH IN ROTATED SORTED ARRAY ===
def search_rotated(nums, target):
    l, r = 0, len(nums)-1
    while l <= r:
        m = (l+r)//2
        if nums[m] == target: return m
        if nums[l] <= nums[m]:
            if nums[l] <= target < nums[m]: r=m-1
            else: l=m+1
        else:
            if nums[m] < target <= nums[r]: l=m+1
            else: r=m-1
    return -1
