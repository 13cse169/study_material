Chapter 01

Foundations of Deep Learning

Core concepts, mathematics, and building blocks that underpin every deep learning system.

01.01

What is Deep Learning?

The theoretical basis and distinction from classical ML

Deep Learning is a subfield of machine learning that uses multi-layered artificial neural networks to learn representations of data with multiple levels of abstraction. Unlike classical ML which requires hand-crafted features, deep learning automatically discovers intricate structure in large datasets.

Machine Learning

Learns from data using algorithms. Requires feature engineering. Works well on structured data with limited size.

Deep Learning

Learns hierarchical feature representations automatically. Excels on unstructured data (images, text, audio) at scale.

Why "Deep"?

Refers to the many hidden layers (depth) in the network. Each layer learns increasingly abstract representations.

Key Characteristics

  • Representation learning: Automatically extracts features without manual engineering
  • Scalability: Performance improves with more data and compute (scaling laws)
  • End-to-end training: Input-to-output pipeline optimized jointly via gradient descent
  • Compositionality: Complex functions built from simpler learned primitives
01.02

Neurons & Perceptrons

The mathematical building block of all neural networks

The Artificial Neuron

An artificial neuron computes a weighted sum of inputs, adds a bias, and passes the result through an activation function.

Output = f( Σ(wᵢ · xᵢ) + b ) where: xᵢ = inputs, wᵢ = weights, b = bias, f = activation function
The Perceptron

The simplest neural network — a single neuron with binary outputs. Used for linearly separable problems.

Learning Rule: wᵢ ← wᵢ + α(y - ŷ)xᵢ where α = learning rate, y = true label, ŷ = predicted label
⚠ Limitation
The XOR problem showed a single perceptron cannot solve non-linearly separable problems — motivating multi-layer networks (MLPs).
Multi-Layer Perceptron (MLP)

An MLP has an input layer, one or more hidden layers, and an output layer. Hidden layers allow learning of non-linear decision boundaries.

Layer output: h⁽ˡ⁾ = f(W⁽ˡ⁾ · h⁽ˡ⁻¹⁾ + b⁽ˡ⁾) Loss minimized via backpropagation + gradient descent
Universal Approximator Fully Connected Feedforward
01.03

Activation Functions

Non-linearities that enable deep networks to learn complex patterns

Without activation functions, a deep network collapses to a single linear transformation. Activations introduce non-linearity, enabling learning of complex mappings.

ACTIVATION · 01
Sigmoid / Logistic
σ(x) = 1 / (1 + e⁻ˣ) Output: (0, 1)
  • Classic binary output activation
  • Suffers from vanishing gradient problem
  • Use: output layer for binary classification
ACTIVATION · 02
Tanh
tanh(x) = (eˣ - e⁻ˣ) / (eˣ + e⁻ˣ) Output: (-1, 1)
  • Zero-centered (better than sigmoid)
  • Still has vanishing gradient issue
  • Use: hidden layers in RNNs
ACTIVATION · 03
ReLU
ReLU(x) = max(0, x) Output: [0, ∞)
  • Most widely used activation function
  • Computationally efficient, no saturation
  • Problem: "dying ReLU" (neurons get stuck at 0)
ACTIVATION · 04
Leaky ReLU / PReLU
Leaky ReLU(x) = max(αx, x), α ≈ 0.01 PReLU: α is a learnable parameter
  • Fixes dying ReLU by allowing small negative slope
  • PReLU learns the optimal α per neuron
ACTIVATION · 05
ELU / SELU
ELU(x) = x if x>0, else α(eˣ-1) SELU = λ · ELU(x) [self-normalizing]
  • Smooth negative values, negative saturation
  • SELU enables self-normalizing deep networks
ACTIVATION · 06
GELU / Swish / Mish
GELU(x) = x·Φ(x) [used in BERT, GPT] Swish(x) = x·σ(x) [used in EfficientNet]
  • Smooth, non-monotonic activations
  • Strong empirical performance in modern LLMs
ACTIVATION · 07
Softmax
Softmax(xᵢ) = eˣⁱ / Σⱼ eˣʲ Output: (0,1), sums to 1
  • Converts logits to probability distribution
  • Use: output layer for multi-class classification

01.04

Backpropagation

The algorithm that makes training deep networks possible

Backpropagation computes gradients of the loss function with respect to all weights using the chain rule of calculus. These gradients are then used by an optimizer to update the weights.

Forward Pass

Data flows through the network layer by layer. Each layer computes activations. The final layer computes the loss.

Loss L = ℓ(ŷ, y) Common losses: MSE: L = (1/n)Σ(yᵢ - ŷᵢ)² CrossEnt: L = -Σ yᵢ·log(ŷᵢ) Hinge: L = max(0, 1 - yᵢ·ŷᵢ)
Backward Pass (Chain Rule)

Gradients are propagated from output back to input through the chain rule:

∂L/∂W⁽ˡ⁾ = ∂L/∂h⁽ˡ⁾ · ∂h⁽ˡ⁾/∂W⁽ˡ⁾ For each layer l, compute: δ⁽ˡ⁾ = (W⁽ˡ⁺¹⁾)ᵀ · δ⁽ˡ⁺¹⁾ ⊙ f'(z⁽ˡ⁾) ← error signal ∂L/∂W⁽ˡ⁾ = δ⁽ˡ⁾ · (h⁽ˡ⁻¹⁾)ᵀ ← weight gradient ∂L/∂b⁽ˡ⁾ = δ⁽ˡ⁾ ← bias gradient
Weight Update
W ← W - α · ∂L/∂W where α = learning rate
Vanishing & Exploding Gradients
In very deep networks, gradients can become tiny (vanish) or huge (explode) as they propagate backward. Solutions: ReLU activations, batch normalization, gradient clipping, residual connections, careful initialization (Xavier/He).
01.05

Optimizers

Algorithms for efficiently minimizing the loss function

OPTIMIZER · 01
SGD (Stochastic Gradient Descent)
w ← w - α · ∇L(w; xᵢ, yᵢ)

Updates weights using a single sample (stochastic) or mini-batch. Simple but slow and noisy.

BaselineSlow Convergence
OPTIMIZER · 02
SGD with Momentum
v ← βv - α·∇L w ← w + v

Accumulates velocity in directions of persistent gradient. Accelerates convergence and dampens oscillations.

Momentum β≈0.9
OPTIMIZER · 03
AdaGrad
G ← G + (∇L)² w ← w - (α/√(G+ε)) · ∇L

Adapts learning rate per parameter. Good for sparse features, but learning rate monotonically decreases.

OPTIMIZER · 04
RMSProp
E[g²] ← ρE[g²] + (1-ρ)(∇L)² w ← w - (α/√(E[g²]+ε)) · ∇L

Fixes AdaGrad's diminishing learning rate using exponential moving average of squared gradients.

RNNs
OPTIMIZER · 05
Adam (Most Popular)
m ← β₁m + (1-β₁)∇L ← 1st moment v ← β₂v + (1-β₂)(∇L)² ← 2nd moment m̂ = m/(1-β₁ᵗ), v̂ = v/(1-β₂ᵗ) ← bias correction w ← w - α·m̂/√(v̂+ε) Defaults: β₁=0.9, β₂=0.999, ε=1e-8

Combines momentum + adaptive LR. The default choice for most deep learning tasks.

⭐ Default ChoiceAdaptive
OPTIMIZER · 06
AdamW
w ← w - α·m̂/√(v̂+ε) - α·λw (decoupled weight decay)

Adam with proper L2 regularization (decoupled weight decay). Default in most LLM training.

01.06

Regularization & Normalization

Techniques to prevent overfitting and stabilize training

L1 Regularization (Lasso)
L_total = L + λΣ|wᵢ|

Produces sparse weights. Drives some weights to exactly zero (feature selection).

L2 Regularization (Ridge/Weight Decay)
L_total = L + λΣwᵢ²

Shrinks all weights toward zero. Prevents large weights, handles multicollinearity.

Dropout
Each neuron deactivated with probability p Typical: p = 0.2–0.5

Randomly zeros neurons during training. Forces redundant representations, acts as ensemble of sub-networks.

Batch Normalization
x̂ = (x - μ_B) / √(σ²_B + ε) y = γx̂ + β

Normalizes layer inputs per mini-batch. Accelerates training, reduces sensitivity to initialization, acts as regularizer.

Layer Normalization
Normalize across features (not batch) LN(x) = γ·(x-μ)/√(σ²+ε) + β

Normalizes across the feature dimension for each example. Preferred in Transformers and RNNs (batch-independent).

Early Stopping

Monitor validation loss during training. Stop when it starts increasing (overfitting begins). Keeps the best checkpoint.

Data Augmentation

Create variations of training samples (flips, rotations, crops, noise, color jitter). Increases effective dataset size without collecting new data.

Weight Initialization
  • Xavier: σ=√(2/(nᵢₙ+nₒᵤₜ)) — sigmoid/tanh
  • He: σ=√(2/nᵢₙ) — ReLU
  • Prevents vanishing/exploding at start

Neural Network Architectures

Specialized network designs for vision, sequences, language, and generation.

02.01

Convolutional Neural Networks (CNN)

The dominant architecture for computer vision tasks

CNNs exploit spatial locality and translation invariance through shared-weight convolutional filters. They automatically learn hierarchical visual features — edges → textures → shapes → objects.

Convolution Operation
Output(i,j) = Σₘ Σₙ Input(i+m, j+n) · Kernel(m,n) Output size = ⌊(N - F + 2P) / S⌋ + 1 where N=input, F=filter, P=padding, S=stride
Pooling Layers
  • Max Pooling: Takes maximum value in each region — preserves most prominent features
  • Average Pooling: Averages values — smoother spatial reduction
  • Global Average Pooling (GAP): Averages entire feature map to single value — replaces FC layers

ResNet — Residual Skip Connections

Residual connections solve the degradation problem in very deep networks by adding a shortcut that bypasses one or more layers.

Output = F(x, {Wᵢ}) + x ← identity shortcut The network learns the residual F(x) instead of the full mapping H(x) Enables training of 100+ layer networks
02.02

RNNs, LSTMs & GRUs

Architectures designed for sequential and temporal data

Recurrent Neural Network (RNN)

RNNs process sequences by maintaining a hidden state that captures information from previous time steps.

hₜ = tanh(Wₓₓxₜ + Wₕₕhₜ₋₁ + b) yₜ = Wᵧₕhₜ + bᵧ Problem: vanishing gradients over long sequences (BPTT)
Long Short-Term Memory (LSTM)

LSTMs solve the vanishing gradient problem using a cell state and three gating mechanisms.

Forget gate: fₜ = σ(Wf·[hₜ₋₁, xₜ] + bf) ← what to forget Input gate: iₜ = σ(Wᵢ·[hₜ₋₁, xₜ] + bᵢ) ← what to update Cell update: C̃ₜ = tanh(Wc·[hₜ₋₁, xₜ] + bc) Cell state: Cₜ = fₜ⊙Cₜ₋₁ + iₜ⊙C̃ₜ ← memory Output gate: oₜ = σ(Wo·[hₜ₋₁, xₜ] + bo) Hidden state: hₜ = oₜ⊙tanh(Cₜ)
LSTM Gates Intuition
Forget gate decides what old memory to erase. Input gate decides what new information to store. Output gate decides what to output from the cell state.
Gated Recurrent Unit (GRU)

GRU is a simpler alternative to LSTM with only two gates and no separate cell state. Fewer parameters, faster training.

Update gate: zₜ = σ(Wz·[hₜ₋₁, xₜ]) Reset gate: rₜ = σ(Wr·[hₜ₋₁, xₜ]) h̃ₜ = tanh(W·[rₜ⊙hₜ₋₁, xₜ]) hₜ = (1-zₜ)⊙hₜ₋₁ + zₜ⊙h̃ₜ
Fewer params than LSTMComparable performance

RNN Variants

Bidirectional RNN

Processes sequence in both directions. Captures context from past and future. Used in NLP encoders.

Seq2Seq

Encoder-decoder architecture. Encoder compresses input sequence, decoder generates output. Used in machine translation.

Attention Mechanism

Allows decoder to focus on relevant encoder states at each step. Precursor to Transformers.

score(hₜ, h̄ₛ) = hₜᵀ·Wₐ·h̄ₛ αₜₛ = softmax(score) context cₜ = Σₛ αₜₛ·h̄ₛ
02.03

Transformers

The revolutionary architecture powering modern AI

Introduced in "Attention Is All You Need" (Vaswani et al., 2017), Transformers replaced RNNs for sequence modeling by relying entirely on self-attention. They enable massive parallelization and long-range dependencies.

Scaled Dot-Product Attention
Q = XWQ, K = XWK, V = XWV ← Query, Key, Value projections Attention(Q, K, V) = softmax(QKᵀ / √dₖ) · V Scaling by √dₖ prevents softmax saturation in high dimensions
Multi-Head Attention

Runs multiple attention heads in parallel, each learning different relationships. Outputs are concatenated and projected.

MultiHead(Q,K,V) = Concat(head₁,...,headₕ)·W° headᵢ = Attention(QWᵢQ, KWᵢK, VWᵢV) Typical: h=8 or 16 heads
Transformer Block
Input x → LayerNorm → Multi-Head Self-Attention → residual add → LayerNorm → Feed-Forward Network (FFN) → residual add Output FFN(x) = max(0, xW₁ + b₁)W₂ + b₂ (2 linear layers + ReLU/GELU)
Positional Encoding

Transformers have no inherent order. Positional encodings inject sequence position information.

Sinusoidal (original): PE(pos, 2i) = sin(pos / 10000^(2i/d)) PE(pos, 2i+1) = cos(pos / 10000^(2i/d)) Learnable: Positional embeddings learned during training (GPT, BERT)

02.04

Generative Adversarial Networks (GANs)

Two-player game for learning generative models

GANs consist of a Generator G and a Discriminator D trained adversarially. G tries to fool D; D tries to distinguish real from fake. At Nash equilibrium, G generates perfect samples.

Min-Max Objective
min_G max_D V(D,G) = 𝔼_{x~p_data}[log D(x)] + 𝔼_{z~p_z}[log(1 - D(G(z)))] Generator loss: Lc = -𝔼[log D(G(z))] Discriminator loss: Ld = -𝔼[log D(x)] - 𝔼[log(1 - D(G(z)))]

GAN Variants

GAN · 01
DCGAN

Uses CNNs for both G and D. Batch norm, strided convolutions, ReLU in G, Leaky ReLU in D. First stable GAN for images.

GAN · 02
WGAN / WGAN-GP

Uses Wasserstein distance instead of JS divergence. More stable training, meaningful loss metric. WGAN-GP uses gradient penalty instead of weight clipping.

GAN · 03
Conditional GAN (cGAN)

Conditions both G and D on class label y. Enables controlled generation: "generate a cat" vs "generate a dog".

GAN · 04
Pix2Pix / CycleGAN

Pix2Pix: paired image translation (sketch→photo). CycleGAN: unpaired image translation using cycle-consistency loss.

GAN · 05
StyleGAN / StyleGAN2

Controls image style at different scales via AdaIN layers. Generates photorealistic faces with controllable attributes. NVIDIA's landmark work.

GAN · 06
BigGAN

Large-scale conditional GAN trained on ImageNet. Uses truncation trick for quality/diversity tradeoff. Class-conditional synthesis at high resolution.

02.05

Autoencoders & VAEs

Unsupervised representation learning and generation

Autoencoder (AE)

Learns compressed latent representation by encoding input to low-dimensional space and decoding back. Trained with reconstruction loss.

Encoder: z = f_θ(x) ← compress to latent space Decoder: x̂ = g_φ(z) ← reconstruct input Loss: L = ||x - x̂||² ← MSE reconstruction loss

Applications: dimensionality reduction, denoising, anomaly detection, compression.

Variational Autoencoder (VAE)

VAE encodes inputs as distributions (μ, σ) rather than points, enabling generative sampling.

Encoder → q_φ(z|x) = N(μ, σ²) ← parameterize distribution Reparameterization: z = μ + σ·ε, ε~N(0,I) ← differentiable sampling Decoder → p_θ(x|z) ELBO Loss: L = E[log p_θ(x|z)] - KL[q_φ(z|x) || p(z)] reconstruction ↑ ↑ regularization (KL divergence)
Latent Space InterpolationGenerativeDisentanglement
Denoising AE

Trained to reconstruct clean input from corrupted version. Forces learning of robust representations.

Sparse AE

Adds L1 penalty on activations, forcing sparse representations. Learns interpretable features.

β-VAE

VAE with β>1 in KL term. Encourages better disentanglement of latent factors.

02.06

Graph Neural Networks (GNN)

Learning on graph-structured data

GNNs extend deep learning to irregular graph structures (molecules, social networks, knowledge graphs) where data has relational connections.

Message Passing Framework
hᵥ⁽ˡ⁺¹⁾ = UPDATE⁽ˡ⁾(hᵥ⁽ˡ⁾, AGGREGATE⁽ˡ⁾({h_u⁽ˡ⁾ : u ∈ N(v)})) Each node aggregates messages from its neighbors, then updates its own representation.
GCN (Graph Convolutional Network)

Spectral convolution on graphs. Updates node features by averaging neighbor features. Basis of most GNN work.

GraphSAGE

Samples fixed-size neighborhood, aggregates with mean/max/LSTM. Scales to large graphs with inductive learning.

GAT (Graph Attention Network)

Applies attention to neighbors — learns which neighbors are most relevant. αᵢⱼ = softmax(a(hᵢ || hⱼ))

Graph Transformer

Applies transformer attention on graphs. Combines global attention with graph structure. State-of-art on many tasks.

Applications
Drug discovery (molecular property prediction), protein folding (AlphaFold uses graph attention), social network analysis, recommendation systems, knowledge graph completion.

03.01

Transfer Learning & Fine-Tuning

Leveraging pre-trained models for new tasks

Transfer learning reuses knowledge from a model trained on a large dataset (source) for a different (target) task, drastically reducing data and compute requirements.

Strategies
Feature Extraction

Freeze all pre-trained layers. Only train a new output head on top. Fast, requires minimal data.

Full Fine-Tuning

Unfreeze all layers and retrain on target task with small learning rate. Requires more data but best performance.

Partial Fine-Tuning

Freeze early layers (low-level features), fine-tune later layers (high-level, task-specific). Common balanced approach.

Parameter-Efficient Fine-Tuning (PEFT)

Fine-tune only a small fraction of parameters, keeping most of the model frozen. Critical for adapting large language models.

  • LoRA (Low-Rank Adaptation): Inject trainable low-rank matrices into attention layers: W' = W + BA, where B∈ℝ^(d×r), A∈ℝ^(r×k), r≪d
  • Prefix Tuning: Prepend learnable "soft prompts" to the input sequence
  • Adapter Layers: Insert small trainable modules between frozen layers
  • QLoRA: Quantize model to 4-bit, apply LoRA — enables LLM fine-tuning on single GPU
03.02

Deep Reinforcement Learning

Learning optimal behavior through trial and interaction

Deep RL combines deep neural networks with reinforcement learning, enabling agents to learn complex policies from raw sensory input (pixels, text).

Core Concepts
Agent selects action aₜ in state sₜ → receives reward rₜ, transitions to sₜ₊₁ Objective: maximize expected cumulative reward Gₜ = Σₖ γᵏrₜ₊ₖ γ ∈ [0,1] = discount factor Policy: π(a|s) = P(action a | state s) Value function: V^π(s) = 𝔼[Gₜ | sₜ=s] Q-function: Q^π(s,a) = 𝔼[Gₜ | sₜ=s, aₜ=a] Advantage: A(s,a) = Q(s,a) - V(s)
DRL · 01
DQN (Deep Q-Network)
Loss = (r + γ·max_a Q(s',a;θ⁻) - Q(s,a;θ))²

Neural network approximates Q-function. Key innovations: experience replay, target network. Achieved superhuman Atari gameplay (DeepMind 2013).

Value-BasedDiscrete Actions
DRL · 02
Policy Gradient / REINFORCE
∇J(θ) = 𝔼[∇log π_θ(a|s)·Gₜ]

Directly optimize policy parameters. High variance but works with continuous actions. Foundation of modern LLM alignment.

Policy-BasedContinuous Actions
DRL · 03
Actor-Critic (A2C/A3C)

Combines policy gradient (actor) with value function (critic). Critic reduces variance of policy gradient updates. A3C uses asynchronous parallel workers.

DRL · 04
PPO (Proximal Policy Optimization)
L = min(rₜ·Â, clip(rₜ, 1-ε, 1+ε)·Â) rₜ = π_θ(a|s)/π_θ_old(a|s)

Clips policy updates to prevent instability. Robust and sample-efficient. Default algorithm in OpenAI's work and RLHF for LLMs.

⭐ RLHFChatGPT
DRL · 05
SAC (Soft Actor-Critic)

Maximum entropy RL framework. Optimizes for both reward and policy entropy (exploration). State-of-art for continuous control tasks (robotics).

AlphaGo: CNN+MCTS+RL defeated world Go champion. AlphaZero: tabula rasa self-play. MuZero: learns its own model of the world.

03.03

Diffusion Models

The state-of-the-art paradigm for generative AI

Diffusion models learn to generate data by reversing a gradual noising process. They have surpassed GANs on image synthesis and are the basis of Stable Diffusion, DALL·E 2, Midjourney, Sora.

Forward Process (Adding Noise)
q(xₜ|xₜ₋₁) = N(xₜ; √(1-βₜ)·xₜ₋₁, βₜ·I) Closed form: q(xₜ|x₀) = N(xₜ; √ᾱₜ·x₀, (1-ᾱₜ)·I) where ᾱₜ = Πₛ₌₁ᵗ (1-βₛ) → At T=1000, xₜ ≈ pure Gaussian noise
Reverse Process (Denoising)
A U-Net ε_θ(xₜ, t) is trained to predict the noise ε added at step t. Loss: L = 𝔼[||ε - ε_θ(√ᾱₜ·x₀ + √(1-ᾱₜ)·ε, t)||²] Sampling: xₜ₋₁ = (1/√αₜ)·(xₜ - βₜ/√(1-ᾱₜ)·ε_θ(xₜ,t)) + σₜz
DDPM

Denoising Diffusion Probabilistic Models. Original formulation by Ho et al. (2020). Slow sampling (1000 steps).

DDIM

Deterministic sampling with fewer steps (50–200). Non-Markovian process enables faster generation with similar quality.

Latent Diffusion (LDM)

Runs diffusion in a compressed latent space (not pixel space). Dramatic efficiency gains. Foundation of Stable Diffusion.

Classifier-Free Guidance

Jointly train conditional and unconditional model. Guidance scale controls quality/diversity tradeoff at inference.

03.04

Large Language Models (LLMs)

Foundation models that power modern AI assistants

LLMs are massive decoder-only transformers trained on trillion-token corpora. They exhibit emergent capabilities — abilities not observed in smaller models — including reasoning, code generation, and in-context learning.

Pre-Training: Next Token Prediction
L = -Σₜ log P(xₜ | x₁,...,xₜ₋₁; θ) Autoregressive language modeling objective KV cache: cache Key/Value tensors for efficient autoregressive generation
RLHF (Reinforcement Learning from Human Feedback)

Aligns LLMs with human preferences. Three-stage pipeline:

  • Stage 1 — SFT: Supervised fine-tuning on human-written demonstrations
  • Stage 2 — RM: Train reward model on human preference comparisons (A vs B)
  • Stage 3 — PPO: Optimize LLM with PPO against reward model signal
LLM objective: 𝔼[r_θ(x,y)] - β·KL[π_θ(y|x) || π_ref(y|x)] maximize reward while staying close to reference policy (KL penalty)
Key LLM Techniques
Flash Attention

IO-aware exact attention implementation. Tiles computation to avoid storing full N×N attention matrix. 2–4× speedup, O(N) memory.

Rotary Position Embedding (RoPE)

Encodes relative position via rotation of Q/K vectors. Enables length generalization. Used in Llama, PaLM.

Grouped-Query Attention (GQA)

Multiple query heads share key/value heads. Reduces KV-cache memory. Used in Llama 3, Gemini.

Mixture of Experts (MoE)

Only a subset of "expert" FFN layers activated per token. Scales model capacity without proportional compute increase. Used in GPT-4, Mixtral.

04.01

Foundations Q&A

🧠

Core Concepts

Backprop, activations, optimization, regularization

Q·01Explain backpropagation step by step. What is the chain rule's role?
+
ANSBackpropagation computes gradients of the loss with respect to every weight using the chain rule. Step 1 — Forward pass: compute activations layer by layer, store intermediate values. Step 2 — Compute loss: measure prediction error with a loss function. Step 3 — Backward pass: starting from the output layer, compute ∂L/∂W at each layer using ∂L/∂W = ∂L/∂a · ∂a/∂z · ∂z/∂W (chain rule). Step 4 — Update weights: W ← W − α·∂L/∂W. The chain rule is essential because the loss depends on weights through a composition of functions — the gradient of a composite function equals the product of individual derivatives.
Q·02What are vanishing and exploding gradients? How do you solve them?
+
ANSVanishing gradients: in deep networks, gradients shrink exponentially as they propagate backward (especially with sigmoid/tanh), causing early layers to learn very slowly. Solutions: use ReLU/GELU activations, residual connections (ResNet), batch normalization, Xavier/He initialization.

Exploding gradients: gradients grow exponentially, causing weight updates to become numerically unstable (NaN/Inf). Solutions: gradient clipping (clip(g, -c, c) or g = g · c/||g||), careful initialization, BatchNorm. In RNNs: use LSTM/GRU gates, which maintain bounded gradient flow via the cell state highway.
Q·03Why is ReLU preferred over sigmoid in hidden layers?
+
ANSThree main reasons: 1. No vanishing gradient: ReLU's derivative is 1 for positive inputs (vs sigmoid's max 0.25), enabling gradient flow in deep networks. 2. Computational efficiency: ReLU is a simple max(0,x) — no exponentials. 3. Sparse activation: roughly half of neurons are inactive (output 0), creating sparse representations that are computationally efficient and often better regularized.

Drawbacks of sigmoid: saturates at extremes → near-zero gradients → slow learning. Also not zero-centered → all-positive gradients → zig-zagging weight updates.
Q·04What is the difference between SGD, Adam, and AdamW?
+
ANSSGD: updates weights using gradient of a mini-batch. Simple, requires tuning learning rate/momentum, can generalize well but converges slowly. Adam: computes adaptive learning rates per parameter using 1st moment (mean of gradients, like momentum) and 2nd moment (mean of squared gradients). Typically faster convergence with less tuning. AdamW: Adam with decoupled weight decay — L2 regularization is applied directly to weights before the gradient update, not incorporated into the gradient. This correctly implements weight decay (standard Adam's weight decay is equivalent to L2 only for SGD). AdamW is the default for training transformers and LLMs.
Q·05What is dropout and why does it work as regularization?
+
ANSDropout randomly zeros out neurons with probability p during training. At inference, all neurons are used and weights scaled by (1-p) (or inverted dropout scales during training). It works because: 1. Ensemble effect: each training step trains a different sub-network; the final model approximates averaging 2ⁿ sub-networks. 2. Prevents co-adaptation: neurons can't rely on specific other neurons being present, forcing redundant representations. 3. Feature independence: encourages each feature to be useful independently. Typical p=0.2–0.5. Note: not used in BatchNorm layers; replaced by attention dropout in Transformers.
Q·06Explain Batch Normalization and Layer Normalization. When to use each?
+
ANSBatch Normalization normalizes across the batch dimension for each feature: μ_B and σ_B computed over mini-batch. Pros: effective regularizer, accelerates training. Cons: depends on batch size (fails with small batches), doesn't work at inference with single examples without running statistics, problematic in RNNs.

Layer Normalization normalizes across the feature dimension for each individual example — independent of batch size. Better for transformers (variable-length sequences), RNNs, and online learning. Standard in all transformer architectures (BERT, GPT). Rule of thumb: CNNs → BatchNorm; Transformers/RNNs → LayerNorm.
Q·07What is the bias-variance tradeoff in deep learning?
+
ANSBias: error from overly simplistic model assumptions (underfitting — high train and test error). Variance: sensitivity to fluctuations in training data (overfitting — low train error, high test error). Total Error = Bias² + Variance + Irreducible Noise.

In deep learning, this classic tradeoff is less rigid due to the double descent phenomenon: as model size increases beyond the interpolation threshold, test error surprisingly decreases again (benign overfitting). Large over-parameterized models can generalize well when trained with gradient descent and proper regularization. Modern practice: use large models + strong regularization (dropout, weight decay, data augmentation).
Q·08What is learning rate scheduling and why is it important?
+
ANSLearning rate scheduling adjusts α during training for better convergence. Common schedules: Step decay — reduce by factor every N epochs. Cosine annealing — α smoothly decays following cosine curve; often with warm restarts. Linear warmup + cosine decay — standard in transformers: ramp LR from 0 to peak over first 1–5% of steps, then cosine decay. One-cycle policy — triangular LR schedule. Why it matters: high LR initially escapes bad local minima; decreasing LR allows fine-tuning convergence to sharp minima. Warmup stabilizes early transformer training when gradients are noisy.

Architecture Q&A

🏗️

CNN, RNN, Transformers, GANs

Architecture-specific concepts and tradeoffs

Q·09What are the advantages of CNNs for images vs MLPs?
+
ANSThree key inductive biases CNNs exploit: 1. Local connectivity: neurons only connect to local receptive fields — nearby pixels are more correlated than distant ones. 2. Parameter sharing / weight tying: the same filter is applied everywhere — learns translation-invariant features (edge detector works regardless of where the edge is). 3. Spatial hierarchy: pooling + conv stacking builds progressively larger receptive fields, learning edges → textures → objects hierarchically.

An MLP treating a 224×224×3 image as a flat vector requires 150,528 input connections per neuron — massive, ignores spatial structure, doesn't generalize. A CNN with 3×3 filters has only 27 weights per filter regardless of image size.
Q·10Why were LSTMs invented? How do they solve the vanishing gradient problem in RNNs?
+
ANSStandard RNNs suffer from vanishing gradients during BPTT (Backpropagation Through Time) because gradients are multiplied by the recurrent weight matrix at every time step — if the largest eigenvalue is <1, gradients exponentially shrink; if >1, they explode.

LSTMs solve this via the cell state — a "memory highway" that runs through time with only additive interactions (not multiplicative). The forget gate can learn to keep Cₜ ≈ Cₜ₋₁ for as many steps as needed, providing a gradient path close to 1.0. The gradient ∂Cₜ/∂Cₜ₋₁ = fₜ (forget gate), which is in (0,1) but the additive structure prevents it from being repeatedly multiplied — unlike vanilla RNN gradients which are products of recurrent weight matrices.
Q·11Explain the Transformer's self-attention mechanism in detail.
+
ANSSelf-attention allows each token to attend to every other token in the sequence. Computation: Each input token x is projected into three vectors: Query (Q=xWQ), Key (K=xWK), Value (V=xWV). Attention scores: Aᵢⱼ = QᵢKⱼᵀ/√dₖ — how relevant is token j to token i? Scaled by √dₖ to prevent softmax saturation for large dimensions. Softmax: normalizes scores to attention weights. Weighted sum: output = softmax(QKᵀ/√dₖ)V.

Why scale by √dₖ? For dₖ-dimensional vectors, the dot product magnitude grows as √dₖ. Without scaling, softmax would be too "peaky" (one dominant attention weight), harming gradient flow.

Multi-head: runs h parallel attention operations with different projections, then concatenates — captures different relationship types simultaneously.
Q·12What is the difference between encoder-only, decoder-only, and encoder-decoder transformers?
+
ANSEncoder-only (BERT): Bidirectional attention — each token attends to all other tokens. Trained with masked language modeling. Best for: classification, NER, question answering, embeddings. Cannot generate text autoregressively.

Decoder-only (GPT): Causal (masked) attention — each token can only attend to previous tokens. Autoregressive generation. Trained with next-token prediction. Best for: text generation, dialogue, in-context learning. Basis of all modern LLMs.

Encoder-Decoder (T5, BART): Encoder processes the input with full attention; decoder attends to encoder output (cross-attention) + previous output tokens (causal self-attention). Best for: machine translation, summarization, seq2seq tasks. More complex but excels when input and output are different sequences.
Q·13What is the GAN training instability problem and how do WGAN/WGAN-GP address it?
+
ANSGAN instabilities: Mode collapse — generator produces limited variety (ignores noise z). Training oscillation — loss oscillates without convergence. Vanishing gradient — when discriminator becomes too good, log(1-D(G(z))) → 0, providing no useful gradient to generator.

Root cause: Original GAN minimizes JS divergence, which is undefined/constant when real and fake distributions don't overlap (early training).

WGAN fix: uses Wasserstein-1 (Earth Mover's) distance instead, which remains meaningful even when distributions don't overlap. Requires critic (discriminator) to be 1-Lipschitz — enforced via weight clipping in WGAN.

WGAN-GP improvement: weight clipping is problematic (causes gradient vanishing/exploding in critic). WGAN-GP adds gradient penalty: λ·(||∇D(x̂)||₂ - 1)² where x̂ is interpolated between real and fake. More stable training, meaningful loss curves, no mode collapse.
Q·14What is the reparameterization trick in VAEs and why is it necessary?
+
ANSThe VAE encoder outputs parameters (μ, σ) of a Gaussian distribution, and we need to sample z~N(μ, σ²) to decode. Problem: sampling is non-differentiable — you can't backpropagate through a random node. Solution — Reparameterization: instead of sampling z directly, sample ε~N(0,I) (independent of parameters) and compute z = μ + σ·ε. Now z is a deterministic function of μ and σ (plus independent noise ε), so gradients can flow through μ and σ during backpropagation. This is the key trick that makes VAEs end-to-end trainable.
Q·15How does ResNet solve the degradation problem? Why does adding more layers without residuals hurt performance?
+
ANSDegradation problem: surprisingly, adding more layers to a plain deep network makes training accuracy worse — not due to overfitting (which would show low train error) but because deep plain networks are hard to optimize. A 56-layer plain network performs worse than a 20-layer one even on training data.

Why it happens: Gradient vanishing/exploding makes weight initialization and optimization extremely sensitive. The network struggles to learn identity mappings through multiple non-linear layers.

ResNet solution: H(x) = F(x) + x. If the optimal transformation is close to identity, the network only needs to learn F(x) ≈ 0 (a small residual), which is easier than learning the full identity mapping H(x) = x through non-linearities. The skip connection also provides a direct gradient path bypassing multiple layers, enabling training of 100–1000+ layer networks.

Transfer Learning, RL, Diffusion, LLMs

Senior-level and research interview questions

Q·16Explain LoRA. How does it reduce trainable parameters during fine-tuning?
+
ANSLoRA (Low-Rank Adaptation) freezes the pre-trained weight matrix W ∈ ℝ^(d×k) and injects a trainable low-rank decomposition: W' = W + BA, where B ∈ ℝ^(d×r) and A ∈ ℝ^(r×k), with rank r ≪ min(d,k). Only B and A are trained (initialized: A~N(0,σ²), B=0 so ΔW=0 at start).

Parameter reduction: original W has d×k parameters; LoRA has r(d+k) ≪ d×k. For d=k=4096 and r=8: original=16.7M params, LoRA=65K (250× reduction). Inference: B·A is merged back into W at inference — zero additional latency. Applied to Q,K,V,O projection matrices in attention. QLoRA extends this to 4-bit quantized base models, enabling 70B model fine-tuning on a single 48GB GPU.
Q·17What is RLHF and why is it important for LLMs?
+
ANSRLHF (Reinforcement Learning from Human Feedback) aligns LLMs to be helpful, harmless, and honest. Why needed: next-token prediction optimizes for statistical likelihood — it can learn to generate harmful, biased, or unhelpful content that is "statistically likely" in training data.

3-step process: (1) SFT — fine-tune base LLM on high-quality human demonstrations. (2) Reward Model — train a classifier on pairs of LLM outputs where humans label which is better; the RM learns to score outputs. (3) PPO — use PPO to maximize expected reward model score while KL-penalizing divergence from SFT model (preventing reward hacking). Result: InstructGPT / ChatGPT. Limitation: reward hacking — LLM can find outputs that score high on RM but are not actually good. Modern alternatives: DPO (Direct Preference Optimization) eliminates the RM and PPO step by directly optimizing on preference data.

Q·19What is Flash Attention and why does it matter?
+
ANSStandard attention computes and stores the N×N attention matrix — O(N²) memory bottleneck that limits context length. Flash Attention (Dao et al., 2022) is an IO-aware implementation that tiles the Q, K, V matrices into blocks and computes attention block-by-block in SRAM (fast GPU cache), never materializing the full N×N matrix in HBM (slow GPU memory).

Result: O(N) memory instead of O(N²). 2–4× training speedup (reduces HBM memory reads/writes, the real bottleneck). Same mathematical output as standard attention — exact, not approximate. Enabled context length scaling from 2K → 100K+ tokens. Flash Attention 2 further optimized thread block partitioning for 2× additional speedup. Critical infrastructure for modern LLMs.
Q·20What is the transformer's computational complexity and what are efficient attention alternatives?
+
ANSStandard self-attention is O(N²·d) time and O(N²) memory where N is sequence length and d is model dimension — quadratic scaling is the fundamental bottleneck for long contexts.

Efficient alternatives: Sparse Attention (Longformer, BigBird): attend only to local window + global tokens → O(N). Linear Attention: kernel trick to factorize attention → O(N). Linformer: project K,V to low-rank (k≪N) → O(N). Performer: random feature approximation of softmax → O(N). State Space Models (Mamba): replaces attention entirely with selective state space — O(N) time, constant memory, can match transformer performance on language tasks with better efficiency at long sequences.
Q·21Explain the difference between model parallelism and data parallelism in LLM training.
+
ANSData Parallelism (DP): full model replicated on each GPU; each GPU processes a different data shard; gradients averaged across GPUs before update. Simple but requires the full model to fit on one GPU.

Tensor Parallelism (TP): split individual weight matrices across GPUs (e.g., attention heads across GPUs). Very fine-grained, high communication overhead within a node.

Pipeline Parallelism (PP): split model layers across GPUs (GPU1 does layers 1–8, GPU2 does layers 9–16...). Micro-batching reduces pipeline bubbles.

3D Parallelism: combine all three (DP×TP×PP) — used in GPT-4 scale training. ZeRO (Zero Redundancy Optimizer): partitions optimizer states, gradients, and parameters across DP ranks — enables data parallelism without memory redundancy. Used in DeepSpeed. Enables training 100B+ parameter models.
Q·22What are emergent abilities in LLMs and what causes them?
+
ANSEmergent abilities are capabilities that appear unpredictably when model scale (parameters × data × compute) crosses certain thresholds — absent in small models but suddenly present in large ones. Examples: multi-step arithmetic, chain-of-thought reasoning, few-shot in-context learning, code generation, theory of mind.

Proposed explanations: (1) Phase transitions: many individual learned capabilities combine nonlinearly at scale. (2) Metric discontinuity: Wei et al. argue some "emergent" abilities reflect discontinuous metrics (accuracy) applied to capabilities that actually improve smoothly. (3) Capability compositionality: complex tasks require multiple sub-skills all to be learned simultaneously; model passes the threshold when all sub-skills are acquired. (4) In-context algorithm learning: transformers may implement gradient-descent-like learning in their forward pass (in-context learning as implicit optimization).
Q·23How do you handle class imbalance in deep learning classification?
+
ANSData-level: oversample minority (SMOTE, augmentation), undersample majority, class-balanced sampling per batch. Loss-level: Weighted cross-entropy — weight classes inversely proportional to frequency (class_weight in PyTorch). Focal Loss (RetinaNet): L = -(1-pₜ)ᵞ·log(pₜ) — down-weights easy (well-classified) examples, focuses learning on hard ones. γ=2 is typical. Excellent for detection tasks. Label smoothing: replaces one-hot labels with soft targets to reduce overconfidence. Evaluation: always use precision/recall/F1/AUC-ROC instead of accuracy for imbalanced data. Architecture: adjust decision threshold at inference (don't default to 0.5 for imbalanced classes).
Q·24What is knowledge distillation and when would you use it?
+
ANSKnowledge distillation trains a small "student" model to mimic a large "teacher" model. The student is trained on soft labels (teacher's probability distributions, temperature-scaled) rather than one-hot ground truth.

L = α·L_CE(y, p_student) + (1-α)·T²·KL(softmax(z_teacher/T), softmax(z_student/T))
Soft labels from teacher carry "dark knowledge" — inter-class relationships (e.g., a cat image being 0.1% similar to a dog) that hard labels don't capture. Temperature T>1 softens these distributions for better transfer.

Use when: deploying on edge devices (mobile, embedded), reducing inference latency, model compression. Examples: DistilBERT (66% fewer params, 97% BERT performance), TinyBERT, MobileNet variants.
Q·25What is contrastive learning and how does it enable self-supervised representation learning?
+
ANSContrastive learning trains a network to pull representations of similar (positive) pairs together and push dissimilar (negative) pairs apart in embedding space — without labels.

SimCLR: augment each image twice (crop+flip+color jitter) → these are positives for each other. All other images in the batch are negatives. InfoNCE loss: maximize cosine similarity of positive pairs vs negatives.

CLIP: pairs images with their captions as positives. Trains image encoder and text encoder jointly. Enables zero-shot transfer: compute similarity between image embedding and text embeddings of class labels.

Why it works: augmentation-invariant representations capture semantically meaningful features (content, not noise) without needing labels. State-of-art representations on ImageNet with no labels, used in foundation models for multi-modal alignment.
