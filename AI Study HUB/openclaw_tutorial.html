HTML Structure

// beginner level · lesson 01

🟢 BEGINNER — Start here if you're new to HTML UI development

📖 THEORY

OpenClaw UI is built on semantic HTML5 with CSS variables for theming. Every OpenClaw project follows a consistent DOM structure:

  • Layout wrapper — root container with CSS variables defined on :root
  • Sidebar — navigation rail, position fixed
  • Main content — offset by sidebar width with margin-left
  • Sections — logically grouped content blocks
boilerplate.html 
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>OpenClaw App</title>
  <style>
    :root {
      --bg: #0a0b0d;
      --accent: #ff5f1f;
      --text: #e8ecf4;
      --surface: #111318;
      --border: #1e2330;
    }
    body {
      background: var(--bg);
      color: var(--text);
      font-family: 'Syne', sans-serif;
    }
  </style>
</head>
<body>

  <aside class="sidebar"><!-- navigation --></aside>
  <main class="main"><!-- content --></main>

</body>
</html>

Navigation Bar

🔵 INTERMEDIATE — Components with state and user interaction

📖 THEORY

<nav class="oc-nav">
  <div class="nav-item active"
       onclick="setNav(this)">Dashboard</div>
  <div class="nav-item"
       onclick="setNav(this)">Projects</div>
</nav>

<script>
function setNav(el) {
  // Remove active from all siblings
  el.parentElement
    .querySelectorAll('.nav-item')
    .forEach(n => n.classList.remove('active'));
  // Set on clicked item
  el.classList.add('active');
}
</script>

Modal Dialogs

// intermediate level · lesson 07

📖 THEORY

Modals are dialogs that overlay content. Key concepts:

  • Overlay — semi-transparent backdrop using fixed positioning
  • Trap focus — keyboard navigation should stay inside the modal
  • Close on overlay click — dismissing by clicking outside is expected UX
  • Animation — use scale + opacity keyframe for smooth entry
LIVE DEMO
modal.html + js 
<div class="overlay" id="modal"
     onclick="closeModal(event)">
  <div class="modal-box">
    <h3>Confirm?</h3>
    <p>Are you sure?</p>
    <button onclick="closeModal()">Close</button>
  </div>
</div>

<script>
function closeModal(e) {
  // Close if clicking overlay itself
  if (!e || e.target.classList.contains('overlay')) {
    document.getElementById('modal')
      .classList.remove('open');
  }
}
</script>

Tabbed Content

// intermediate level · lesson 10

📖 THEORY

Tabs allow switching between content panels without a page reload. The tab list uses an underline indicator (not a box) on the active tab. Match tab labels to panel IDs with data-* attributes for clean JS targeting. Use aria-selected for accessibility.

tabs.js 
function switchTab(el, containerId) {
  const parent = document.getElementById(containerId);
  
  // Remove active from all tabs
  parent.querySelectorAll('.oc-tab')
    .forEach(t => t.classList.remove('active'));
  el.classList.add('active');
  
  // Hide all panels, show the target
  parent.querySelectorAll('.oc-tab-content')
    .forEach(c => c.style.display = 'none');
  document.getElementById(el.dataset.tab)
    .style.display = 'block';
}

Skeleton Loaders

// advanced level · lesson 13

📖 THEORY

Skeleton screens replace spinners for content-heavy pages. They mirror the shape of real content, reducing perceived loading time. The shimmer effect is achieved with an animated background-position on a gradient — no JS required. Replace skeletons with real content once the data resolves.

LIVE DEMO — Loading state

  background: linear-gradient(90deg,
    #111318 25%,
    #181c24 50%,
    #111318 75%
  );
  background-size: 200% 100%;
  animation: shimmer 1.5s infinite;
  border-radius: 6px;
}

@keyframes shimmer {
  0%   { background-position: 200% 0; }
  100% { background-position: -200% 0; }

Toast Notifications

// advanced level · lesson 14

📖 THEORY

Toasts are non-blocking temporary notifications. Key advanced patterns:

  • Auto-dismiss — use setTimeout to hide after 3 seconds
  • Queue management — prevent overlap with a notification queue array
  • Fixed positioning — always render in a corner, outside the document flow
  • Slide-in animation — translateY + opacity for smooth entrance
LIVE DEMO
toast.js 
function showToast(message, type = 'success') {
  const toast = document
    .getElementById('toast');
  
  toast.textContent = message;
  toast.className = 'oc-toast show ' + type;
  
  // Auto-dismiss after 3 seconds
  setTimeout(() => {
    toast.classList.remove('show');
  }, 3000);
}

Form Validation

// advanced level · lesson 15

📖 THEORY

Advanced form validation combines real-time feedback (on blur or input events) with submit-time validation. OpenClaw pattern: validate on blur event, show inline error messages below each field, prevent submit if isValid === false. Consider regex for email/phone validation.

LIVE DEMO
⚠ Enter a valid email address

validation.js 
function validateEmail(input) {
  const re = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  const valid = re.test(input.value);
  const err = document
    .getElementById('emailError');

  if (!valid && input.value) {
    input.style.borderColor = '#ef4444';
    err.style.display = 'block';
  } else {
    input.style.borderColor = '#22c55e';
    err.style.display = 'none';
  }
}

Dynamic Theming

// advanced level · lesson 16

📖 THEORY

OpenClaw's theme system uses CSS Custom Properties on the :root element. Switching themes is as simple as overriding these variables. Store the user's preference in localStorage and apply on page load. For dark/light, add a data-theme attribute on <html> and use attribute selectors in CSS.

theming.css + js 
/* Define themes with attribute selectors */
:root { --bg: #0a0b0d; --text: #e8ecf4; }
[data-theme="light"] {
  --bg: #ffffff;
  --text: #111318;
  --surface: #f4f6fa;
  --border: #dde1f0;
}

// JS theme switcher
function setTheme(theme) {
  document.documentElement
    .setAttribute('data-theme', theme);
  localStorage.setItem('theme', theme);
}

// On load — restore preference
const saved = localStorage.getItem('theme');
if (saved) setTheme(saved);

Interview Questions

// 25 questions · beginner to advanced

🔶 INTERVIEW PREP — Common questions asked in frontend UI developer roles
1
What is the difference between id and class in HTML?
▼
id is unique — only one element per page can have a given id. It's used for JS targeting (getElementById) and anchor links. class can be shared across multiple elements and is used for styling groups. IDs have higher CSS specificity than classes.
  • Use id for unique elements: modals, sections
  • Use class for reusable component styles
  • In CSS: #id specificity = 100, .class = 10
2
What are CSS Custom Properties (variables) and why use them?
▼
CSS variables (declared with --name: value on :root) store reusable values. They are:
  • Dynamic — can be changed at runtime with JavaScript
  • Cascading — can be overridden in child scopes
  • Theming — change one variable to retheme the entire UI
Example: color: var(--accent); — if you change --accent once, all elements update automatically.
3
What is the Box Model in CSS?
▼
Every HTML element is a rectangular box with four layers — from inside out: content → padding → border → margin.
  • content: actual text/image area
  • padding: inner spacing between content and border
  • border: visible frame around the element
  • margin: outer spacing from other elements
Use box-sizing: border-box (recommended) so that padding and border are included inside the declared width.
4
What is semantic HTML and why does it matter?
▼
Semantic HTML uses elements that convey meaning — <header>, <nav>, <main>, <section>, <article>, <footer> vs generic <div>. Benefits:
  • Accessibility: screen readers navigate by landmarks
  • SEO: search engines understand page structure
  • Maintainability: code is self-documenting
5
Explain the difference between display: block, inline, and inline-block.
▼
  • block: takes full width, starts on new line. Examples: div, p, h1
  • inline: takes only content width, flows with text. Cannot set width/height. Examples: span, a, strong
  • inline-block: flows with text BUT allows width/height. Best of both worlds. Used for buttons.
6
What is CSS Flexbox and when would you use it?
▼
Flexbox is a 1-dimensional layout model for arranging items in a row or column. Use it for:
  • Navigation bars and button groups
  • Centering elements vertically and horizontally
  • Evenly distributing items with justify-content: space-between
  • Responsive card rows that wrap with flex-wrap: wrap
Key properties: display:flex, flex-direction, justify-content, align-items, gap.
7
What is CSS specificity and how is it calculated?
▼
Specificity determines which CSS rule wins when multiple rules target the same element:
  • Inline styles: 1000
  • IDs (#id): 100
  • Classes, attributes, pseudo-classes (.class, [attr], :hover): 10
  • Elements and pseudo-elements (div, ::before): 1
!important overrides all, but should be avoided. Higher specificity = winning rule.
8
What is event bubbling and how do you stop it?
▼
Event bubbling means when an event fires on a child element, it propagates up to all parent elements. Example: clicking a button inside a modal also "clicks" the modal overlay. Stop it with event.stopPropagation() on the child's handler. Use event.target to check which element was actually clicked and conditionally respond.
9
What is the difference between null and undefined in JavaScript?
▼
  • undefined: a variable declared but not yet assigned a value. Default for function params not passed.
  • null: intentionally empty/no value. Must be explicitly assigned.
  • typeof null === 'object' — this is a JavaScript bug retained for backwards compatibility.
  • Use strict equality: null === null is true, but null == undefined is also true (loose).
10
How does position: fixed differ from position: absolute?
▼
  • fixed: positioned relative to the viewport. Stays in place when the page scrolls. Used for headers, sidebars, modals.
  • absolute: positioned relative to the nearest positioned ancestor (with position: relative/absolute/fixed). Scrolls with the page if no fixed ancestor.
11
What is a CSS media query? Write one for mobile.
▼
Media queries apply CSS rules conditionally based on device properties (viewport width, orientation, resolution). Mobile-first approach:

@media (max-width: 768px) { .sidebar { display: none; } }

Common breakpoints: 480px (mobile), 768px (tablet), 1024px (desktop). Use min-width for mobile-first, max-width for desktop-first.
12
What is localStorage vs sessionStorage vs cookies?
▼
  • localStorage: persists across sessions, no expiry, ~5MB, same-origin only
  • sessionStorage: cleared when tab is closed, per-tab, ~5MB
  • cookies: sent with every HTTP request, can set expiry, smaller (~4KB), can be HttpOnly (not accessible to JS)
Use localStorage for theme/preferences. Use cookies for auth tokens (with HttpOnly + Secure flags).
13
What is the difference between == and === in JavaScript?
▼
== performs type coercion before comparison. === checks both value AND type (strict equality).
  • 0 == false → true (coercion converts false to 0)
  • 0 === false → false (different types: number vs boolean)
  • "5" == 5 → true | "5" === 5 → false
Always prefer === to avoid unexpected coercion bugs.
14
What is event delegation and why is it useful?
▼
Event delegation attaches ONE event listener to a parent element instead of many listeners on each child. It leverages event bubbling — clicks on children bubble up to the parent listener.

Use event.target inside the handler to identify which child was clicked. This is much more efficient for dynamic lists where children are added/removed at runtime.
15
How do you implement a CSS-only toggle (no JavaScript)?
▼
Use a hidden <input type="checkbox"> with a paired <label>. Clicking the label toggles the checkbox state. CSS responds via the :checked pseudo-class:

input:checked + .track { background: var(--accent); }

The sibling combinator + selects the element immediately after the checked input. No JS required for visual toggling.
16
What is the difference between async/await and Promises?
▼
Both handle asynchronous operations. async/await is syntactic sugar built on top of Promises, making async code look synchronous:
  • Promise: fetch(url).then(r => r.json()).then(data => ...)
  • async/await: const data = await fetch(url).then(r => r.json())
Use try/catch with async/await for error handling instead of .catch().
17
What is CSS Grid vs Flexbox — when to use each?
▼
  • Flexbox: 1D layout (row OR column). Best for navigation bars, button groups, centering one item.
  • CSS Grid: 2D layout (rows AND columns simultaneously). Best for full-page layouts, dashboards, card grids.
Rule of thumb: if you need columns AND rows at the same time → Grid. If you're just arranging items in one direction → Flexbox. They can (and should) be combined.
18
What is debouncing and throttling? Give UI examples.
▼
Both limit how often a function is called:
  • Debounce: waits until a pause in calls. Fires AFTER the last call. Use for search input — only query API when user stops typing for 300ms.
  • Throttle: fires at most once per time window, regardless of how often called. Use for scroll events, window resize — cap at 60fps.
Example: const search = debounce(queryAPI, 300); — called on every keystroke, but queryAPI only runs 300ms after the last keystroke.
19
How do you make a website accessible (a11y)?
▼
Key accessibility practices:
  • Use semantic HTML landmarks (<main>, <nav>, <header>)
  • All images need alt text
  • Forms need associated <label> elements
  • Interactive elements must be keyboard focusable (visible :focus ring)
  • Color contrast: WCAG AA requires 4.5:1 ratio for normal text
  • Use aria-* attributes for dynamic components (modals, tabs)
  • Never use outline: none without a custom focus style
20
What is the Critical Rendering Path?
▼
The sequence of steps the browser takes to render a page:
  • 1. Parse HTML → build DOM
  • 2. Parse CSS → build CSSOM
  • 3. Combine DOM + CSSOM → Render Tree
  • 4. Layout (calculate element positions/sizes)
  • 5. Paint (draw pixels to screen)
  • 6. Composite (layer ordering)
Optimize by: inlining critical CSS, deferring non-critical JS, using font-display: swap, and minimizing layout-triggering properties.
21
What is a CSS animation? Explain @keyframes.
▼
@keyframes defines animation steps. Attach with animation property:

@keyframes shimmer { 0% { opacity: 0; } 100% { opacity: 1; } }
.el { animation: shimmer 1s ease infinite; }

Properties: animation-duration, animation-timing-function (ease/linear), animation-delay, animation-iteration-count, animation-fill-mode (forwards = stay at end state).
22
How do you implement dark/light mode in CSS?
▼
Two approaches:
  • CSS media query: @media (prefers-color-scheme: dark) { ... } — respects OS setting
  • Data attribute: [data-theme="dark"] { --bg: #000; } — user-controlled via JS toggle
Best practice: define all colors as CSS variables on :root, then override the variables in the dark theme block. Never hardcode color values in component styles.
23
What is z-index and what is a stacking context?
▼
z-index controls the stack order of positioned elements — higher value = on top. But it only works on elements with position other than static.

A stacking context is an isolated z-index scope. Created by: position + z-index, opacity < 1, transform, filter. Elements inside a stacking context cannot escape it — a z-index of 9999 inside a context with z-index 1 cannot appear above a context with z-index 2.
24
What are Web Performance metrics (LCP, CLS, FID/INP)?
▼
Google's Core Web Vitals:
  • LCP (Largest Contentful Paint): time for largest visible element to load. Target: <2.5s. Optimize: preload hero images, use CDN.
  • CLS (Cumulative Layout Shift): measures unexpected layout shifts. Target: <0.1. Avoid: images without dimensions, injected content above text.
  • INP (Interaction to Next Paint): measures responsiveness to user input. Target: <200ms. Optimize: reduce JS blocking, use requestAnimationFrame.
25
How would you build a reusable component system in vanilla HTML/CSS/JS?
▼
Key principles for a component system:
  • CSS naming convention: BEM (.block__element--modifier) or a prefix system like oc-* to namespace components
  • CSS variables: define tokens on :root for colors, spacing, typography
  • JS factory functions: function createButton(label, variant) { ... return element; }
  • Web Components: use customElements.define('oc-button', ...) for native browser components with shadow DOM encapsulation
  • Template literals: generate HTML strings and insert with innerHTML for simple cases
