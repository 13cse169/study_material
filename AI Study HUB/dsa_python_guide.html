Understanding how algorithms scale with input size — the most fundamental skill in DSA.
1
What is Big-O Notation?
Definition

Big-O notation describes the upper bound of an algorithm's time or space requirements as the input size (n) grows toward infinity. It tells us the worst-case performance, ignoring constants and lower-order terms.

Python
# O(1) - Constant time
def get_first(arr):
    return arr[0]  # Always 1 operation

# O(n) - Linear time
def find_max(arr):
    max_val = arr[0]
    for x in arr:      # n operations
        if x > max_val:
            max_val = x
    return max_val

# O(n²) - Quadratic time
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):         # n iterations
        for j in range(n-i-1):  # n iterations each
            if arr[j] > arr[j+1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]

Two Pointer Technique
Definition

Use two pointers (left & right) moving toward each other or in the same direction to solve problems in O(n) that would naively take O(n²).

Python — Two Sum (Sorted Array)
def two_sum_sorted(arr, target):
    """Find pair that sums to target — O(n) time, O(1) space"""
    left, right = 0, len(arr) - 1
    
    while left < right:
        current_sum = arr[left] + arr[right]
        
        if current_sum == target:
            return (left, right)   # Found!
        elif current_sum < target:
            left += 1               # Need larger sum
        else:
            right -= 1              # Need smaller sum
    
    return None

# Example
arr = [1, 3, 5, 7, 9]
print(two_sum_sorted(arr, 12))  # (2, 4) → arr[2]+arr[4]=14? No → (1,4)=10? Try (2,3)=12 ✓

Sliding Window
Definition

Maintain a window of elements, expand or shrink it to satisfy conditions. Converts O(n²) brute force to O(n) for subarray/substring problems.

Python — Max Sum Subarray of size K
def max_sum_subarray(arr, k):
    """Max sum of subarray of size k — O(n)"""
    n = len(arr)
    if n < k:
        return -1
    
    # Compute sum of first window
    window_sum = sum(arr[:k])
    max_sum = window_sum
    
    # Slide the window
    for i in range(k, n):
        window_sum += arr[i] - arr[i - k]  # Add new, remove old
        max_sum = max(max_sum, window_sum)
    
    return max_sum

print(max_sum_subarray([2, 1, 5, 1, 3, 2], 3))  # 9 (5+1+3)

Longest Substring Without Repeating Characters
Classic Problem

Find the length of the longest substring without repeating characters. Uses sliding window + hash set for O(n) solution.

Python — Sliding Window
def length_of_longest_substring(s):
    char_set = set()
    left = 0
    max_len = 0
    
    for right in range(len(s)):
        # Shrink window until no duplicate
        while s[right] in char_set:
            char_set.remove(s[left])
            left += 1
        
        char_set.add(s[right])
        max_len = max(max_len, right - left + 1)
    
    return max_len

print(length_of_longest_substring("abcabcbb"))  # 3 ("abc")
print(length_of_longest_substring("pwwkew"))     # 3 ("wke")

Python — Complete Implementation
class Node:
    def __init__(self, val):
        self.val = val
        self.next = None

class LinkedList:
    def __init__(self):
        self.head = None
    
    def append(self, val):           # O(n)
        new_node = Node(val)
        if not self.head:
            self.head = new_node
            return
        curr = self.head
        while curr.next:
            curr = curr.next
        curr.next = new_node
    
    def prepend(self, val):          # O(1)
        new_node = Node(val)
        new_node.next = self.head
        self.head = new_node
    
    def delete(self, val):           # O(n)
        if not self.head: return
        if self.head.val == val:
            self.head = self.head.next
            return
        curr = self.head
        while curr.next and curr.next.val != val:
            curr = curr.next
        if curr.next:
            curr.next = curr.next.next
    
    def reverse(self):               # O(n)
        prev = None
        curr = self.head
        while curr:
            nxt = curr.next
            curr.next = prev
            prev = curr
            curr = nxt
        self.head = prev
    
    def display(self):
        curr = self.head
        while curr:
            print(curr.val, end=" → ")
            curr = curr.next
        print("None")

Floyd's Cycle Detection (Fast & Slow Pointers)
Algorithm

Use two pointers: slow moves 1 step, fast moves 2 steps. If there's a cycle, they will eventually meet. If fast reaches None, no cycle exists.

Python — Detect Cycle
def has_cycle(head):
    slow, fast = head, head
    
    while fast and fast.next:
        slow = slow.next        # Move 1 step
        fast = fast.next.next  # Move 2 steps
        
        if slow == fast:
            return True          # Cycle detected!
    
    return False                  # No cycle

# Find middle node
def find_middle(head):
    slow, fast = head, head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
    return slow  # slow is at middle

Two fundamental abstract data types with different ordering principles.
1
Stack — Last In, First Out (LIFO)
Definition

A stack is an ADT where elements are added (push) and removed (pop) from the same end — the top. Think of a stack of plates: you always add and remove from the top.

Stack — TOP is always popped first
30
20
10

push(30) → pop() returns 30 → LIFO order

Python — Stack Implementation
# Method 1: Using Python list (simplest)
stack = []
stack.append(10)   # push → O(1)
stack.append(20)
stack.append(30)
stack.pop()         # pop → O(1), returns 30
stack[-1]          # peek → O(1), returns 20

# Method 2: Custom Stack class
class Stack:
    def __init__(self):
        self.data = []
    
    def push(self, item):  self.data.append(item)
    def pop(self):         return self.data.pop()
    def peek(self):        return self.data[-1]
    def is_empty(self):   return len(self.data) == 0
    def size(self):        return len(self.data)

# Classic application: Valid Parentheses
def is_valid(s):
    stack = []
    mapping = {")": "(", "}": "{", "]": "["}
    for char in s:
        if char in mapping:
            top = stack.pop() if stack else "#"
            if mapping[char] != top:
                return False
        else:
            stack.append(char)
    return not stack

Queue — First In, First Out (FIFO)
Definition

Elements are enqueued at the rear and dequeued from the front. Like a line at a store — first person in line is first served.

Python — Queue with deque
from collections import deque

queue = deque()

queue.append("B")
queue.append("C")
queue.popleft()         # dequeue → O(1), returns "A"
queue[0]               # front → "B"

# BFS uses a Queue
def bfs(graph, start):
    visited = set([start])
    queue = deque([start])
    result = []
    
    while queue:
        node = queue.popleft()
        result.append(node)
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)
    return result
1
Deque (Double-Ended Queue)
Definition

Supports O(1) insertion and deletion at both ends. Combines properties of stack and queue.

Python — Sliding Window Maximum
from collections import deque

def max_sliding_window(nums, k):
    """Find max in each window of size k — O(n)"""
    dq = deque()   # stores indices
    result = []
    
    for i, num in enumerate(nums):
        # Remove out-of-window indices
        while dq and dq[0] < i - k + 1:
            dq.popleft()
        
        # Remove smaller elements (useless)
        while dq and nums[dq[-1]] < num:
            dq.pop()
        
        dq.append(i)
        
        if i >= k - 1:
            result.append(nums[dq[0]])  # Max is at front
    
    return result

Binary Trees & BST
Hierarchical data structures with a root node and subtrees.
1
Binary Tree Structure
Definition

A tree where each node has at most 2 children (left and right). The topmost node is the root. Nodes with no children are leaves.

Binary Search Tree — Left < Root < Right
Python — BST Implementation
class TreeNode:
    def __init__(self, val):
        self.val = val
        self.left = self.right = None

class BST:
    def insert(self, root, val):    # O(h) h=height
        if not root: return TreeNode(val)
        if val < root.val:
            root.left = self.insert(root.left, val)
        else:
            root.right = self.insert(root.right, val)
        return root
    
    def search(self, root, val):    # O(h)
        if not root or root.val == val: return root
        if val < root.val: return self.search(root.left, val)
        return self.search(root.right, val)

# Tree Traversals
def inorder(root):     # Left → Root → Right (sorted for BST)
    return inorder(root.left) + [root.val] + inorder(root.right) if root else []

def preorder(root):    # Root → Left → Right
    return [root.val] + preorder(root.left) + preorder(root.right) if root else []

def postorder(root):   # Left → Right → Root
    return postorder(root.left) + postorder(root.right) + [root.val] if root else []

# BFS Level Order
from collections import deque
def level_order(root):
    if not root: return []
    result, queue = [], deque([root])
    while queue:
        level = []
        for _ in range(len(queue)):
            node = queue.popleft()
            level.append(node.val)
            if node.left: queue.append(node.left)
            if node.right: queue.append(node.right)
        result.append(level)
    return result

# Two Sum with HashMap — O(n)
def two_sum(nums, target):
    seen = {}  # val → index
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i

print(two_sum([2, 7, 11, 15], 9))  # [0, 1]

Python — Merge Sort & Quick Sort
# Merge Sort — O(n log n), stable
def merge_sort(arr):
    if len(arr) <= 1:
        return arr
    
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    
    return merge(left, right)

def merge(left, right):
    result = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i]); i += 1
        else:
            result.append(right[j]); j += 1
    return result + left[i:] + right[j:]

# Quick Sort — O(n log n) avg
def quick_sort(arr, lo=0, hi=None):
    if hi is None: hi = len(arr) - 1
    if lo >= hi: return
    pivot = partition(arr, lo, hi)
    quick_sort(arr, lo, pivot - 1)
    quick_sort(arr, pivot + 1, hi)

def partition(arr, lo, hi):
    pivot = arr[hi]
    i = lo - 1
    for j in range(lo, hi):
        if arr[j] <= pivot:
            i += 1
            arr[i], arr[j] = arr[j], arr[i]
    arr[i+1], arr[hi] = arr[hi], arr[i+1]
    return i + 1

Searching Algorithms
Linear search for unsorted data; binary search for sorted data.
1
Binary Search
Definition

Repeatedly halve the search space by comparing the target with the middle element. Requires a sorted array. O(log n) time.

Python — Binary Search Templates
# Classic Binary Search
def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    
    while left <= right:
        mid = left + (right - left) // 2  # Avoids overflow
        
        if arr[mid] == target:
            return mid      # Found!
        elif arr[mid] < target:
            left = mid + 1  # Search right half
        else:
            right = mid - 1 # Search left half
    
    return -1  # Not found

# Find first/last occurrence
def find_first(arr, target):
    left, right, result = 0, len(arr)-1, -1
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == target:
            result = mid; right = mid - 1  # Keep searching left
        elif arr[mid] < target: left = mid + 1
        else: right = mid - 1
    return result

# Search in rotated sorted array
def search_rotated(nums, target):
    left, right = 0, len(nums) - 1
    while left <= right:
        mid = (left + right) // 2
        if nums[mid] == target: return mid
        if nums[left] <= nums[mid]:  # Left sorted
            if nums[left] <= target < nums[mid]: right = mid - 1
            else: left = mid + 1
        else:                        # Right sorted
            if nums[mid] < target <= nums[right]: left = mid + 1
            else: right = mid - 1
    return -1

Recursion
Functions that call themselves to solve smaller subproblems.
1
Recursion Fundamentals
Definition

Recursion solves a problem by breaking it into smaller instances of the same problem. Every recursive function needs: (1) Base case — stops recursion, (2) Recursive case — calls itself with smaller input.

Python — Classic Recursive Problems
# Factorial — O(n)
def factorial(n):
    if n <= 1: return 1       # Base case
    return n * factorial(n-1) # Recursive case

# Fibonacci — O(2ⁿ) naive, O(n) with memo
def fib(n, memo={}):
    if n <= 1: return n
    if n not in memo:
        memo[n] = fib(n-1, memo) + fib(n-2, memo)
    return memo[n]

# Power — O(log n)
def power(base, exp):
    if exp == 0: return 1
    if exp % 2 == 0:
        half = power(base, exp//2)
        return half * half       # Square the half
    return base * power(base, exp-1)

# Backtracking — Generate all permutations
def permutations(nums):
    result = []
    def backtrack(current, remaining):
        if not remaining:
            result.append(current[:])
            return
        for i in range(len(remaining)):
            current.append(remaining[i])
            backtrack(current, remaining[:i] + remaining[i+1:])
            current.pop()  # Undo choice
    backtrack([], nums)
    return result

Graph Representation & Traversal
Definition

A graph G = (V, E) consists of vertices V and edges E. Can be directed or undirected, weighted or unweighted. Represented as adjacency list (sparse) or matrix (dense).

Python — BFS, DFS, Dijkstra
from collections import deque
import heapq

graph = {
    'A': ['B', 'C'],
    'B': ['A', 'D', 'E'],
    'C': ['A', 'F'],
    'D': ['B'], 'E': ['B'], 'F': ['C']
}

# BFS — Shortest path (unweighted)
def bfs(graph, start):
    visited = {start}
    queue = deque([start])
    order = []
    while queue:
        node = queue.popleft()
        order.append(node)
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)
    return order

# DFS — Recursive
def dfs(graph, node, visited=None):
    if visited is None: visited = set()
    visited.add(node)
    print(node, end=" ")
    for neighbor in graph[node]:
        if neighbor not in visited:
            dfs(graph, neighbor, visited)

# Dijkstra's — Shortest path weighted
def dijkstra(graph, start):
    dist = {node: float('inf') for node in graph}
    dist[start] = 0
    heap = [(0, start)]
    
    while heap:
        d, u = heapq.heappop(heap)
        if d > dist[u]: continue
        for v, w in graph[u]:
            if dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                heapq.heappush(heap, (dist[v], v))
    return dist

Dynamic Programming
Break problems into overlapping subproblems, store results to avoid recomputation.
1
DP Fundamentals + Classic Problems
Definition

Dynamic Programming = Recursion + Memoization (top-down) or Tabulation (bottom-up). Applicable when a problem has: (1) Optimal substructure, (2) Overlapping subproblems.

Python — Knapsack, LCS, Coin Change
# 0/1 Knapsack — O(n*W)
def knapsack(weights, values, W):
    n = len(weights)
    dp = [[0] * (W+1) for _ in range(n+1)]
    
    for i in range(1, n+1):
        for w in range(W+1):
            # Don't take item i
            dp[i][w] = dp[i-1][w]
            # Take item i if it fits
            if weights[i-1] <= w:
                dp[i][w] = max(dp[i][w],
                    dp[i-1][w-weights[i-1]] + values[i-1])
    return dp[n][W]

# Longest Common Subsequence — O(m*n)
def lcs(s1, s2):
    m, n = len(s1), len(s2)
    dp = [[0]*(n+1) for _ in range(m+1)]
    for i in range(1, m+1):
        for j in range(1, n+1):
            if s1[i-1] == s2[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1
            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])
    return dp[m][n]

# Coin Change — O(amount * coins)
def coin_change(coins, amount):
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0
    for coin in coins:
        for x in range(coin, amount + 1):
            dp[x] = min(dp[x], dp[x - coin] + 1)
    return dp[amount] if dp[amount] != float('inf') else -1

1
Min Heap with heapq
Definition

A Min-Heap is a complete binary tree where every parent node ≤ its children. Python's heapq module implements a min-heap. For max-heap, negate values.

Python — heapq & K-th Largest
import heapq

# Min-Heap
heap = []
heapq.heappush(heap, 5)
heapq.heappush(heap, 1)
heapq.heappush(heap, 3)
heapq.heappop(heap)   # Returns 1 (minimum)

# Build heap from list — O(n)
nums = [3, 1, 4, 1, 5, 9]
heapq.heapify(nums)

# K-th Largest Element — O(n log k)
def kth_largest(nums, k):
    min_heap = []
    for num in nums:
        heapq.heappush(min_heap, num)
        if len(min_heap) > k:
            heapq.heappop(min_heap)  # Remove smallest
    return min_heap[0]  # Root = k-th largest

# Top K frequent elements

Tries (Prefix Trees)
Tree structure for efficient string prefix operations.
1
Trie Implementation
Definition

A Trie (prefix tree) stores strings where each node represents a character. Enables O(m) search, insert, and prefix-lookup where m is the word length. Ideal for autocomplete and spell-check.

Python — Trie Implementation
class TrieNode:
    def __init__(self):
        self.children = {}
        self.is_end = False

class Trie:
    def __init__(self):
        self.root = TrieNode()
    
    def insert(self, word):         # O(m)
        node = self.root
        for char in word:
            if char not in node.children:
                node.children[char] = TrieNode()
            node = node.children[char]
        node.is_end = True
    
    def search(self, word):         # O(m)
        node = self.root
        for char in word:
            if char not in node.children:
                return False
            node = node.children[char]
        return node.is_end
    
    def starts_with(self, prefix):  # O(m)
        node = self.root
        for char in prefix:
            if char not in node.children:
                return False
            node = node.children[char]
        return True

# Usage
trie = Trie()
trie.insert("apple")
trie.search("apple")     # True
trie.starts_with("app") # True

#001
Two Sum
Easy
Given an array of integers nums and a target integer, return indices of two numbers that add up to target. Each input has exactly one solution; don't use the same element twice.
Example: nums=[2,7,11,15], target=9 → [0,1]
Python — O(n) Solution
def two_sum(nums, target):
    seen = {}  # value → index
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i
💡Use a hash map to store seen values. For each element, check if complement exists — O(n) vs O(n²) brute force.
#002
Reverse a Linked List
Easy
Given the head of a singly linked list, reverse the list and return the reversed list's head.
Example: 1→2→3→4→5 → 5→4→3→2→1
Python — Iterative O(n)
def reverse_list(head):
    prev = None
    curr = head
    while curr:
        nxt = curr.next   # Save next
        curr.next = prev   # Reverse link
        prev = curr        # Move prev forward
        curr = nxt         # Move curr forward
    return prev            # New head
#003
Valid Parentheses
Easy
Given a string containing only '(', ')', '{', '}', '[' and ']', determine if the input string is valid. Open brackets must be closed by the same type and in the correct order.
Python — Stack O(n)
def is_valid(s):
    stack = []
    pairs = {')':'(', '}':'{', ']':'['}
    for c in s:
        if c in pairs:
            if not stack or stack[-1] != pairs[c]:
                return False
            stack.pop()
        else:
            stack.append(c)
    return not stack
#004
Maximum Subarray (Kadane's Algorithm)
Medium
Find the contiguous subarray (containing at least one number) which has the largest sum and return its sum.
Example: [-2,1,-3,4,-1,2,1,-5,4] → 6 (subarray [4,-1,2,1])
Python — Kadane's O(n)
def max_subarray(nums):
    max_sum = curr_sum = nums[0]
    for num in nums[1:]:
        # Either extend or start fresh
        curr_sum = max(num, curr_sum + num)
        max_sum = max(max_sum, curr_sum)
    return max_sum
💡Key insight: if current sum becomes negative, it's better to start fresh from the next element.

#006
Number of Islands
Medium
Given an m×n 2D grid of '1's (land) and '0's (water), count the number of islands. An island is surrounded by water and formed by connecting adjacent lands horizontally or vertically.
Python — DFS O(m*n)
def num_islands(grid):
    if not grid: return 0
    count = 0
    
    def dfs(r, c):
        if r < 0 or r >= len(grid) or \
           c < 0 or c >= len(grid[0]) or \
           grid[r][c] != '1': return
        grid[r][c] = '#'   # Mark visited
        dfs(r+1,c); dfs(r-1,c)
        dfs(r,c+1); dfs(r,c-1)
    
    for r in range(len(grid)):
        for c in range(len(grid[0])):
            if grid[r][c] == '1':
                dfs(r, c)
                count += 1
    return count

Interview Q&A
Top DSA interview questions from FAANG and top tech companies — with detailed answers.
Q
What is the difference between an Array and a Dynamic Array (list in Python)?
▼
A static array has a fixed size defined at creation. A dynamic array (Python list) automatically resizes — when full, it allocates a larger buffer (typically 2x), copies elements, then continues. This makes append O(1) amortized, though occasional resizes are O(n). Python lists are dynamic arrays under the hood, implemented in C with over-allocation for performance.
Q
How do you find a duplicate in an array of n+1 integers in range [1,n] using O(1) space?
▼
Use Floyd's cycle detection (fast-slow pointer). Treat array values as "next pointers" — since a value repeats, it creates a cycle. Start both slow and fast at index 0. Move slow by 1, fast by 2. When they meet, reset slow to 0 and move both by 1 — where they meet again is the duplicate. Time: O(n), Space: O(1).
Python
def find_duplicate(nums):
    slow = fast = nums[0]
    while True:
        slow = nums[slow]
        fast = nums[nums[fast]]
        if slow == fast: break
    slow = nums[0]
    while slow != fast:
        slow = nums[slow]
        fast = nums[fast]
    return slow
Q
Explain the sliding window technique and when to use it.
▼
Sliding window maintains a contiguous subarray/substring using two pointers. Use it when asked for: longest/shortest subarray satisfying a condition, fixed-size window aggregates, or substring problems. Types: (1) Fixed window — advance both pointers together. (2) Variable window — expand right, shrink left when condition violated. Converts O(n²) brute force to O(n).
Q
How would you rotate an array by k positions to the right in O(1) space?
▼
Use the three-reversal trick: (1) Reverse the entire array, (2) Reverse first k elements, (3) Reverse remaining n-k elements. Each reversal is O(n) and O(1) space. Remember to handle k = k % n to avoid unnecessary rotations when k ≥ n.
Q
How do you detect and remove a cycle in a linked list?
▼
Detect cycle with Floyd's algorithm (slow/fast pointers). To find the start of cycle: after detection, reset one pointer to head and move both one step at a time — they meet at cycle start. To remove: find node just before cycle start and set its .next = None.
Q
How do you find the k-th node from the end of a linked list in one pass?
▼
Use two pointers: advance the first pointer k steps ahead, then move both until the first pointer reaches the end. The second pointer is now at the k-th from last. This is O(n) time with one pass and O(1) space — no need to know the list length first.
Q
When would you choose a linked list over an array?
▼
Choose linked list when: frequent insertions/deletions at the beginning (O(1) vs O(n)), unknown or highly variable size, implementing stacks/queues. Choose array when: random access needed (O(1) indexing), cache performance matters (contiguous memory), or mostly traversal with few modifications.
Q
What is the difference between BFS and DFS on a tree? When to use each?
▼
BFS (level-order) uses a queue and explores all nodes at each depth before going deeper. Use for: shortest path, level-by-level processing, finding nearest neighbor. DFS (uses stack/recursion) explores as far as possible before backtracking. Use for: path finding, cycle detection, topological sort, tree height. DFS uses O(h) space, BFS uses O(w) where w = max width.
Q
How do you check if a binary tree is balanced?
▼
A balanced tree has height difference ≤ 1 for every node's subtrees. Use bottom-up DFS: return -1 if unbalanced (height difference > 1), otherwise return the actual height. This runs in O(n) vs the O(n²) top-down approach.
Python
def is_balanced(root):
    def height(node):
        if not node: return 0
        L = height(node.left)
        R = height(node.right)
        if L == -1 or R == -1 or abs(L-R) > 1:
            return -1
        return max(L, R) + 1
    return height(root) != -1
Q
What is the Lowest Common Ancestor (LCA) problem and how do you solve it?
▼
LCA of two nodes p and q is the deepest node that is an ancestor of both. Algorithm: if root is None, p, or q — return root. Recurse on both subtrees. If both return non-null, current root is the LCA. If only one returns non-null, return that. Time: O(n), Space: O(h).
Q
How do you identify if a problem can be solved with Dynamic Programming?
▼
Look for two properties: (1) Optimal substructure — optimal solution can be built from optimal solutions of subproblems. (2) Overlapping subproblems — same subproblems are solved multiple times. Common DP keywords: "maximum/minimum ways", "count number of ways", "is it possible to...". Start by identifying the state (what changes between subproblems) and the transition (how to compute from smaller states).
Q
Explain Dijkstra's algorithm and its limitations.
▼
Dijkstra finds shortest path from a source to all vertices in a weighted graph using a min-heap. At each step, extract the unvisited vertex with minimum distance, then relax its neighbors. Time: O((V+E) log V) with a binary heap. Limitation: does NOT work with negative edge weights (use Bellman-Ford instead, which is O(VE)).
Q
What is topological sort and when is it used?
▼
Topological sort orders nodes in a Directed Acyclic Graph (DAG) such that for every edge u→v, u comes before v. Used for: task scheduling, build systems (Makefile), course prerequisites. Implemented via DFS (add to stack after visiting all neighbors) or Kahn's BFS algorithm (repeatedly remove nodes with in-degree 0). Only possible on DAGs.
Q
What is the difference between time and space complexity?
▼
Time complexity measures how execution time grows with input size. Space complexity measures how memory usage grows. Both use Big-O notation. There's often a space-time tradeoff — e.g., memoization uses extra space O(n) to reduce time from O(2ⁿ) to O(n). Always discuss both in interviews and justify your tradeoff choice.
Q
What is amortized analysis? Give an example.
▼
Amortized analysis considers the average cost per operation over a sequence of operations, even if some individual operations are expensive. Example: Python list.append() is usually O(1), but occasionally O(n) when resizing. Amortized over n appends: total work is O(n), so amortized O(1) per append. The cost of expensive operations is "spread" over the cheap ones.
Q
Explain the difference between a tree and a graph.
▼
A tree is a special case of a graph: it's a connected, acyclic, undirected graph with exactly N-1 edges for N nodes. Every tree is a graph, but not vice versa. Trees have a hierarchical structure with a root. Graphs can have cycles, multiple paths between nodes, disconnected components, and directed or undirected edges.
Q
What is hashing? How does a hash table handle collisions?
▼
A hash function maps a key to an index in an array. Collisions (two keys mapping to same index) are handled by: (1) Chaining — each slot holds a linked list of entries. (2) Open addressing — probe next slot (linear, quadratic, or double hashing). Python's dict uses open addressing with a compact table. Load factor (entries/capacity) affects performance; Python resizes at 2/3 capacity.
Q
What is a heap and how is it different from a BST?
▼
Heap: complete binary tree with heap property (parent ≥ children for max-heap). Guarantees O(1) find-max, O(log n) insert/delete. BST: each node > all left subtree, < all right subtree. Guarantees O(log n) search, insert, delete (balanced). Use heap when you only need repeated min/max access. Use BST for ordered traversal and range queries.
