Projects & Deployment · 8 Questions
Intermediate
Q01How would you deploy a deep learning model to production?
mid+
Production deployment pipeline: (1) Model optimisation: Quantise (INT8/INT4), prune, or distil for smaller/faster inference. Export to ONNX for framework-agnostic serving. (2) Serving: FastAPI/Flask for REST API; Triton Inference Server for high-throughput GPU serving; vLLM for LLM inference (PagedAttention). (3) Containerisation: Docker image with GPU support (CUDA base image). (4) Scaling: Kubernetes for orchestration, auto-scaling based on queue depth. (5) Monitoring: Track latency, throughput, prediction distribution drift (Evidently), GPU utilisation. (6) CI/CD: Automated testing of model performance on each new version.
Q02How do you evaluate a generative model?
hard+
Evaluation depends on the task: Image generation: FID (Fréchet Inception Distance) — lower=better, compares feature distribution of real vs generated images. IS (Inception Score) — higher=better. Human evaluation. Text generation: Perplexity, BLEU (for translation), ROUGE (for summarisation), BERTScore (semantic), Human preference (via A/B testing). Chatbots: MT-Bench, AlpacaEval, LMSYS Chatbot Arena (human ELO ratings). Code generation: Pass@k — fraction of problems solved if k attempts allowed (HumanEval). None of these perfectly capture quality — human evaluation remains the gold standard.
Advanced
Q03How would you approach debugging a model that isn't converging?
hard+
Systematic debugging: (1) Overfit a single batch — if loss doesn't go to zero on 1 batch, there's a code bug. (2) Check gradient flow — log grad norms per layer: if any layer's grad is zero, ReLU dying or no connection. (3) Learning rate — run LR range test, plot loss vs LR. (4) Data sanity check — print random batches, check label distribution, verify normalisation. (5) Loss on random init — cross-entropy loss should ≈ log(n_classes) initially. (6) Architecture — remove complexity (BatchNorm, Dropout) temporarily. (7) Precision — check for NaN/Inf in activations (register forward hooks). (8) Reduce dataset to 1K samples and verify it overfits.
