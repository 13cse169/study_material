THE PERCEPTRON

The atomic unit of every neural network
⚡Single Neuron

A neuron computes a weighted sum of its inputs, adds a bias, then applies an activation function:

z = w₁x₁ + w₂x₂ + ... + wₙxₙ + b = Wᵀx + b a = σ(z) ← activation function applied to pre-activation z

Weights W — learned parameters that scale each input's importance
Bias b — learned offset, allows the neuron to activate even when inputs are zero
Activation σ — introduces non-linearity (without it, deep networks = 1 linear layer)

🧠From Neuron to Network

Stack neurons in layers. Each layer transforms the previous layer's output:

Input Layer
x ∈ ℝⁿ
→
Hidden Layer 1
a¹=σ(W¹x+b¹)
→
Hidden Layer 2
a²=σ(W²a¹+b²)
→
Output Layer
ŷ=softmax(Wᴸaᴸ⁻¹)
💡Universal Approximation Theorem: A neural network with one hidden layer and enough neurons can approximate any continuous function to arbitrary precision. Width + depth → expressive power.
🔢Parameter Count
Computing Parameters
# Network: 784 → 256 → 128 → 10
layer1_params = 784 * 256 + 256   # weights + bias = 200,960
layer2_params = 256 * 128 + 128   # = 32,896
layer3_params = 128 * 10  + 10    # = 1,290
total = layer1_params + layer2_params + layer3_params
print(f"Total: {total:,} parameters")  # 235,146

# PyTorch one-liner
from torchinfo import summary
summary(model, input_size=(1, 784))

ACTIVATION FUNCTIONS

The source of non-linearity in neural networks
📊Why Activation Functions?

Without non-linear activations, stacking linear layers = one big linear layer: W₃(W₂(W₁x)) = Wx. You'd lose all representational power of depth. Activations break this linearity.

⚡Activation Functions Compared

💀Dying ReLU Problem & Solutions

If a ReLU neuron's input is always negative, its output is always 0, gradient is always 0, weights never update — the neuron is "dead". Typically affects 10–40% of neurons in naively initialised networks.

PyTorch · Activation Functions
import torch.nn as nn

# Standard choices
relu     = nn.ReLU()          # Fast, default
leaky    = nn.LeakyReLU(0.01) # Fixes dying ReLU

# Simple MLP with GELU
model = nn.Sequential(
    nn.Linear(784, 256), nn.GELU(),
    nn.Linear(256, 128), nn.GELU(),
    nn.Linear(128, 10)

FORWARD PASS

Data flows through the network: input → prediction
➡️Step-by-Step Forward Pass
1

Input

Flatten/normalise input x. For MNIST: 28×28 image → 784-dim vector, scaled to [0,1].

2

Linear Transform Layer 1

z¹ = W¹x + b¹ → pre-activation

3

Activation

a¹ = ReLU(z¹) → kills negatives

4

Repeat for L layers

zˡ = Wˡaˡ⁻¹ + bˡ, aˡ = σ(zˡ)

5

Output

ŷ = softmax(Wᴸaᴸ⁻¹ + bᴸ) → class probabilities summing to 1

NumPy Forward Pass from Scratch
import numpy as np

def relu(z): return np.maximum(0, z)
def softmax(z):
    e = np.exp(z - z.max(-1, keepdims=True))  # numerically stable
    return e / e.sum(-1, keepdims=True)

def forward(x, params):
    W1,b1,W2,b2,W3,b3 = params
    a1 = relu(x  @ W1 + b1)
    a2 = relu(a1 @ W2 + b2)
    out = softmax(a2 @ W3 + b3)
    return out  # shape: (batch, 10)

BACKPROPAGATION

How neural networks learn from their mistakes
🔙The Algorithm
1

Forward Pass

Compute and cache all intermediate activations a¹, a², ..., aᴸ and pre-activations z¹...zᴸ.

2

Compute Loss

L = CrossEntropy(ŷ, y). Scalar value measuring prediction error.

3

Output Gradient

δᴸ = ∂L/∂zᴸ = ŷ - y (for softmax + cross-entropy — conveniently simple!)

4

Backpropagate

For each layer l from L-1 to 1: δˡ = ((Wˡ⁺¹)ᵀ δˡ⁺¹) ⊙ σ'(zˡ)

5

Compute Gradients

∂L/∂Wˡ = δˡ · (aˡ⁻¹)ᵀ, ∂L/∂bˡ = δˡ

6

Update Parameters

W ← W - η · ∂L/∂W for each layer

💻Backprop from Scratch (NumPy)
2-Layer Backprop
def backward(x, y, params, cache):
    W1,b1,W2,b2 = params
    a1, z1, a2, z2 = cache
    m = x.shape[0]  # batch size

    # Output layer delta (softmax + cross-entropy)
    delta2 = a2 - y                        # (m, 10)

    # Gradients for layer 2
    dW2 = a1.T @ delta2 / m
    db2 = delta2.mean(0)

    # Backprop through ReLU
    delta1 = (delta2 @ W2.T) * (z1 > 0)   # ReLU gradient

    # Gradients for layer 1
    dW1 = x.T @ delta1 / m
    db1 = delta1.mean(0)

    return dW1, db1, dW2, db2

INTERVIEW Q&A

NN Fundamentals · 12 Questions
Beginner
Q01What is an activation function and why do we need it?
easy+
An activation function introduces non-linearity into the network. Without it, stacking linear layers gives: W₃(W₂(W₁x)) = (W₃W₂W₁)x — just one linear transformation, no matter how deep the network. Non-linearity allows the network to learn complex, non-linear decision boundaries. ReLU is the default for hidden layers — fast, simple, doesn't saturate for positive values. Sigmoid/Tanh used for gates (LSTM) and output layers (binary classification). Softmax for multi-class output — converts logits to probability distribution.
Q02What is the difference between batch, stochastic, and mini-batch gradient descent?
easy+
Batch GD: Compute gradient over all training samples. Accurate but slow (one update per epoch). Memory intensive — may not fit dataset in RAM. Stochastic GD (SGD): One sample per update. Very noisy gradients, many updates per epoch. The noise can help escape local minima but convergence is erratic. Mini-batch GD: Standard in practice. Batches of 32–512. Benefits: (1) vectorisation makes it GPU-efficient; (2) batch noise provides regularisation; (3) frequent updates for fast convergence. Batch size is a hyperparameter — larger batches = smoother gradients but less regularisation.
Q03Explain overfitting and how to prevent it.
easy+
Overfitting occurs when a model learns the training data too well — including noise — and fails to generalise to unseen data (high train accuracy, low val accuracy). Prevention: Dropout — randomly zero out activations during training (p=0.1–0.5), forcing the network to learn redundant representations. L2 regularisation — penalises large weights. Data augmentation — artificially expand training set. Early stopping — stop when validation loss stops improving. Batch normalisation — acts as implicit regulariser. Reduce model capacity — fewer parameters. Best practice: start with simpler model, add capacity only when underfitting.
Intermediate
Q04How does Adam optimizer work? Why is it better than SGD?
mid+
Adam (Adaptive Moment Estimation) maintains two moving averages per parameter: m = β₁m + (1-β₁)g (first moment, like momentum) and v = β₂v + (1-β₂)g² (second moment, variance of gradients). Bias-corrected: m̂ = m/(1-β₁ᵗ), v̂ = v/(1-β₂ᵗ). Update: θ -= η · m̂/(√v̂ + ε). The adaptive learning rate means: parameters with consistently large gradients get smaller updates; rare/small gradients get larger updates. This is especially beneficial for sparse features in NLP. β₁=0.9, β₂=0.999, ε=1e-8 are almost always kept as defaults. However, for vision tasks with proper tuning, SGD+momentum often outperforms Adam.
Q05What is dropout and how does it work at inference?
mid+
During training, dropout randomly sets activations to 0 with probability p, and scales remaining ones by 1/(1-p) (inverted dropout) so expected value stays the same. This prevents co-adaptation — neurons can't rely on specific other neurons always being present. Effect: trains an ensemble of 2^n different network architectures. At inference, dropout is turned off (model.eval() in PyTorch) and all neurons participate — equivalent to averaging the ensemble. The key implementation detail: PyTorch's nn.Dropout handles the 1/(1-p) scaling during training, so inference needs no adjustment.
Q06Why is weight initialisation important?
mid+
If weights are all zeros: all neurons compute identical outputs and receive identical gradients — symmetry breaking never happens, the network never learns. If weights are too large: activations saturate (sigmoid→1, ReLU very large), gradients vanish or explode. If too small: activations shrink layer by layer, gradients vanish. Proper initialisation (He for ReLU, Xavier for sigmoid/tanh) keeps the variance of activations and gradients approximately constant across layers, allowing training to begin from a stable regime. Modern networks also rely on BatchNorm/LayerNorm to maintain this stability as training progresses.
Advanced
Q07Explain the exploding gradient problem and gradient clipping.
hard+
In deep networks (especially RNNs), gradients can grow exponentially through backprop — if weight matrices have eigenvalues >1, repeated multiplication explodes them. Symptoms: loss becomes NaN, parameters become Inf. Gradient clipping rescales the global gradient norm if it exceeds a threshold: if ||g|| > max_norm, g = g · (max_norm / ||g||). PyTorch: torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0). This clips the magnitude but preserves direction. Applied after loss.backward() but before optimizer.step(). Common thresholds: 1.0 for transformers, 5.0 for RNNs.
Q08What is the difference between model.train() and model.eval() in PyTorch?
hard+
model.train() enables training behaviour: Dropout randomly zeros activations; BatchNorm uses batch statistics (μ, σ from current batch) and updates running stats. model.eval() disables training behaviour: Dropout becomes identity; BatchNorm uses running mean/var accumulated during training (more stable, deterministic). Always use model.eval() with torch.no_grad() during validation/inference — no_grad() skips gradient tracking, saving memory and computation (~50% faster). Forgetting to switch modes is a common bug causing train/eval accuracy discrepancies.
