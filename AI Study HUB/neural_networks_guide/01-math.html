Vectors · Matrices · Operations · Eigenvalues
📐Scalars, Vectors, Matrices, Tensors

Deep learning is tensor computation. Every piece of data — an image, a sentence, a batch — is a tensor flowing through your network.

ObjectRankExampleDL Use

✖️Matrix Multiplication — The Core Operation

Every linear layer computes: Y = XW + b. This is a matrix multiply. For X∈ℝ^(m×n) and W∈ℝ^(n×p), result Y∈ℝ^(m×p).

Y = XW + b (m×n) · (n×p) + (p,) = (m×p) Inner dimensions must match!
Python · NumPy
import numpy as np

X = np.random.randn(32, 784)   # batch of 32 images, 784 pixels
W = np.random.randn(784, 256)  # weight matrix
b = np.zeros(256)               # bias

Y = X @ W + b                    # @ is matrix multiply → shape (32, 256)
print(Y.shape)                   # (32, 256)
🔄Key Operations for DL

Transpose

Flip rows ↔ cols. A^T[i,j] = A[j,i]. Used in attention: QKᵀ

Dot Product

a·b = Σaᵢbᵢ = |a||b|cosθ. Measures similarity. Foundation of attention scores.

Norms

L1: Σ|xᵢ|, L2: √(Σxᵢ²). Used in regularisation (weight decay = L2 norm penalty).

Eigendecomposition

Av = λv. Eigenvectors give principal directions; eigenvalues give importance. Core of PCA.

🌀Singular Value Decomposition (SVD)

Any matrix M = UΣVᵀ. U and V are orthogonal, Σ is diagonal with singular values. Used in PCA, matrix factorisation, and model compression (low-rank approximation of weight matrices).

NumPy SVD
M = np.random.randn(100, 50)
U, S, Vt = np.linalg.svd(M, full_matrices=False)
# Low-rank approximation with k=10 components
k = 10
M_approx = U[:, :k] @ np.diag(S[:k]) @ Vt[:k, :]
print(f"Compression: {M.size} → {k*(U.shape[0]+Vt.shape[1])} params")

CALCULUS

Derivatives · Chain Rule · Gradients · Jacobians
📈Why Calculus? Gradient Descent!

Training a neural network = minimising a loss function L(θ) by updating parameters θ. We do this by computing the gradient ∂L/∂θ and stepping opposite to it.

θ ← θ − η · ∂L/∂θ η = learning rate (step size) ∂L/∂θ = gradient (direction of steepest ascent)
🔗Chain Rule — Backpropagation's Engine

The chain rule lets us differentiate composite functions. A neural network is a composition of functions: L(f_n(f_{n-1}(...f_1(x)))). Backprop applies chain rule layer by layer:

If y = f(g(x)), then dy/dx = (df/dg) · (dg/dx) For network: ∂L/∂W¹ = ∂L/∂a² · ∂a²/∂z² · ∂z²/∂a¹ · ∂a¹/∂z¹ · ∂z¹/∂W¹
💡Key insight: Gradients flow backwards through the network in exact reverse order. Each layer receives ∂L/∂output and returns ∂L/∂input and ∂L/∂weights.
🎯Partial Derivatives & Gradients

The gradient ∇L is a vector of partial derivatives: ∇L = [∂L/∂w₁, ∂L/∂w₂, ..., ∂L/∂wₙ]. It points in the direction of steepest increase of L.

PyTorch Autograd
import torch

# Automatic differentiation example
x = torch.tensor([2.0, 3.0], requires_grad=True)
y = x[0]**2 + 3*x[1]   # y = x₀² + 3x₁

y.backward()
print(x.grad)  # tensor([4., 3.])  → [2x₀, 3]
📐Jacobians and Hessians

Jacobian Matrix

Generalises gradient to vector-valued functions. J[i,j] = ∂yᵢ/∂xⱼ. Used in backprop through multi-output layers.

Hessian Matrix

Second-order derivatives. H[i,j] = ∂²L/∂wᵢ∂wⱼ. Tells curvature. Used in Newton's method and saddle point analysis.

PROBABILITY

Distributions · Bayes · Entropy · MLE
🎲Why Probability in DL?

Neural networks output probability distributions. Softmax gives P(class|input). Loss functions like cross-entropy are derived from maximum likelihood estimation. Understanding probability is essential for understanding why models work.

📊Key Distributions
DistributionUse in DLParameters
BernoulliBinary classification outputp
CategoricalMulti-class softmax outputp₁...pₖ
Gaussian (Normal)Weight init, VAE latent spaceμ, σ²
UniformXavier/Glorot inita, b
🔑Cross-Entropy Loss — Derived from MLE

Maximum Likelihood Estimation: find parameters θ that maximise P(data|θ). For classification, this becomes minimising cross-entropy:

L = -Σ yᵢ · log(ŷᵢ) = -log(ŷ_correct_class) (for one-hot y) This is exactly -log P(correct class | input, θ)
📌Cross-entropy loss = negative log-likelihood of the correct label under the model's predicted distribution. Minimising loss = maximising likelihood.
ℹ️Information Theory — KL Divergence & Entropy
Entropy: H(P) = -Σ P(x) log P(x) KL Divergence: D_KL(P||Q) = Σ P(x) log(P(x)/Q(x)) Cross-Entropy: H(P,Q) = H(P) + D_KL(P||Q)

KL divergence measures how different distribution Q is from reference P. Used in VAEs (KL loss term) and knowledge distillation.

INTERVIEW Q&A

Math Foundation · 10 Questions · All Levels
Beginner
Q01Why do we need matrix multiplication in neural networks?
easy+
Every linear layer in a neural network performs a matrix multiplication: Y = XW + b. This is a linear transformation that maps the input feature space to a new representation space. The weight matrix W learns which combinations of input features are useful for the task. Matrix multiplication is highly parallelisable on GPUs, which is why neural networks train so efficiently on modern hardware. Without matrix operations, we'd need loops — far slower.
Q02What is the chain rule and why is it fundamental to backpropagation?
easy+
The chain rule states that if y = f(g(x)), then dy/dx = f'(g(x)) · g'(x). A neural network is a composition of many functions (layers). Backpropagation is simply the chain rule applied recursively from the loss back to the first layer. Each layer receives the upstream gradient (∂L/∂output) and computes two things: (1) the gradient for its parameters (∂L/∂W = ∂L/∂output · ∂output/∂W), and (2) the gradient for the previous layer (∂L/∂input = ∂L/∂output · ∂output/∂input). This is efficient because we compute each intermediate gradient exactly once.
Q03What is gradient descent and its variants?
easy+
Batch GD: Uses entire dataset per update. Accurate gradients but slow and memory-intensive. Stochastic GD (SGD): One sample per update. Very noisy but fast and can escape local minima. Mini-batch GD: Standard practice — use batches of 32–512 samples. Balances accuracy and efficiency. SGD with momentum: Accumulates velocity v = βv + g, update θ -= η·v. Reduces oscillation. Adam: Adaptive learning rates using first moment (momentum) and second moment (variance). Most popular in practice.
Intermediate
Q04Why is cross-entropy preferred over MSE for classification?
mid+
MSE loss with sigmoid/softmax outputs causes the vanishing gradient problem. When the prediction is very wrong (sigmoid output ≈ 0 when target=1), MSE gradient ∂L/∂z = (ŷ-y)·σ'(z) is nearly zero because σ'(z) → 0. Cross-entropy loss -log(ŷ) has gradient ∂L/∂z = (ŷ-y) — the sigmoid cancels out, giving a large gradient when predictions are wrong. This means faster learning. Additionally, cross-entropy is the proper probabilistic loss derived from maximum likelihood estimation for categorical distributions.
Q05What is the difference between L1 and L2 regularisation?
mid+
L2 (Weight Decay): Adds λΣwᵢ² to loss. Gradient: 2λwᵢ — pushes weights towards zero proportionally. Results in small but non-zero weights. Probabilistically equivalent to Gaussian prior on weights. Implemented in PyTorch as weight_decay in optimiser. L1 (Lasso): Adds λΣ|wᵢ| to loss. Gradient: λ·sign(wᵢ) — constant penalty regardless of weight magnitude. Promotes sparsity — many weights become exactly zero. Useful for feature selection. In practice, L2 is used almost universally in DL; L1 or ElasticNet for sparse features.
Q06Explain KL divergence and where it's used in deep learning.
mid+
KL divergence D_KL(P||Q) = Σ P(x)log(P(x)/Q(x)) measures how much information is lost when using Q to approximate P. It's asymmetric: D_KL(P||Q) ≠ D_KL(Q||P). In deep learning: VAEs use KL divergence to regularise the latent space, ensuring the encoder output (approximate posterior) stays close to the prior N(0,I). The ELBO = reconstruction loss + KL divergence. Knowledge Distillation minimises KL between student and teacher softmax distributions. Policy Gradient methods (PPO) clip the KL between old and new policies for stability.
Advanced
Q07What is the vanishing gradient problem and how do modern architectures solve it?
hard+
In deep networks with sigmoid/tanh activations, gradients get multiplied by σ'(z) at each layer during backprop. Since max(σ'(z)) = 0.25, gradients shrink by ~4x per layer — exponentially small in early layers. Solutions: ReLU activations — gradient is 1 for z>0, doesn't shrink. Residual connections (skip connections) — gradient has a direct path of magnitude 1 bypassing layers: ∂L/∂x = ∂L/∂(x+F(x)) = ∂L/∂output · (1 + ∂F/∂x). The "+1" means gradient always reaches early layers. Batch/Layer Norm — normalises activations, keeping them in the linear regime of activations. Careful initialisation (He, Xavier) — keeps gradient variance roughly constant across layers.
Q08Why does batch normalisation work? What are its effects on training?
hard+
BatchNorm normalises activations within a mini-batch: x̂ = (x-μ_B)/σ_B, then scales: y = γx̂ + β. Effects: (1) Reduces internal covariate shift — each layer's input distribution stays stable, so deeper layers don't need to constantly readjust to upstream changes. (2) Allows higher learning rates — loss landscape is smoother. (3) Acts as regulariser — stochastic batch statistics add noise. (4) Reduces sensitivity to weight initialisation. Limitations: doesn't work well with small batch sizes; problematic in RNNs (use LayerNorm instead); train/inference discrepancy (running mean/var used at test time).
Q09Explain the concept of eigenvalues in the context of neural network training.
hard+
The Hessian matrix H = ∂²L/∂θ² captures the curvature of the loss landscape. Its eigenvalues determine training stability: if the largest eigenvalue λ_max is large, the loss changes rapidly in that direction — using a learning rate η > 2/λ_max causes divergence. This is why learning rate scheduling is critical. Saddle points — have positive and negative eigenvalues — are more common than local minima in high-dimensional loss surfaces. SGD noise helps escape saddle points. Conditioning: a ratio of max to min eigenvalue (condition number) >> 1 means directions have very different curvatures, causing slow convergence — Adam addresses this with per-parameter adaptive rates.
Q10What is Xavier/Glorot and He initialisation? When to use each?
hard+
Xavier/Glorot: W ~ Uniform(-√(6/(nᵢₙ+nₒᵤₜ)), √(6/(nᵢₙ+nₒᵤₜ))). Derived assuming linear activations. Keeps variance of activations and gradients equal across layers. Use with sigmoid or tanh. He/Kaiming: W ~ N(0, 2/nᵢₙ). Accounts for ReLU's zero-ing of negative half (effectively halves the variance). Use with ReLU, Leaky ReLU, ELU. Intuition: if variance grows layer-by-layer → exploding activations; if it shrinks → vanishing. Good initialisation starts training from a regime where forward and backward passes have stable variance.
