ЁЯФ╣Why RNNs?

Standard feed-forward networks assume inputs are independent. Text, audio, time-series have temporal dependencies тАФ what happened before matters. RNNs maintain a hidden state hтВЬ that summarises past information.

ЁЯФ╣LSTM Gates

LSTM adds three gates that control information flow:

fтВЬ = ╧Г(Wf┬╖[hтВЬтВЛтВБ,xтВЬ] + bf) тЖР Forget gate: what to erase iтВЬ = ╧Г(Wi┬╖[hтВЬтВЛтВБ,xтВЬ] + bi) тЖР Input gate: what to write g╠ГтВЬ = tanh(Wg┬╖[hтВЬтВЛтВБ,xтВЬ] + bg) тЖР Candidate cell values cтВЬ = fтВЬтКЩcтВЬтВЛтВБ + iтВЬтКЩg╠ГтВЬ тЖР Cell state update oтВЬ = ╧Г(Wo┬╖[hтВЬтВЛтВБ,xтВЬ] + bo) тЖР Output gate hтВЬ = oтВЬтКЩtanh(cтВЬ) тЖР Hidden state
Python ┬╖ LSTM for Sequence Classification
class LSTMClassifier(nn.Module):
    def __init__(self, vocab_size, embed_dim, hidden, n_classes):
        super().__init__()
        self.embed = nn.Embedding(vocab_size, embed_dim, padding_idx=0)
        self.lstm  = nn.LSTM(embed_dim, hidden, batch_first=True,
                              bidirectional=True, dropout=0.3)
        self.fc    = nn.Linear(hidden*2, n_classes)

    def forward(self, x):
        x = self.embed(x)          # (B, T, E)
        out, _ = self.lstm(x)      # (B, T, 2H) bidirectional
        pooled = out.mean(dim=1)   # global average pool over time
        return self.fc(pooled)

Q01What is the vanishing gradient problem in RNNs?
mid+
In vanilla RNNs, gradients are backpropagated through time: тИВL/тИВhтВБ = тИВL/тИВhтВЬ ┬╖ тИПс╡втВМтВВс╡Ч (тИВhс╡в/тИВhс╡втВЛтВБ). Each term involves WтВХтВХ ┬╖ tanh'(zс╡в). Since max(tanh') = 1 and weights are typically <1, this product shrinks exponentially тАФ gradients in early timesteps тЖТ 0. The network can't learn long-range dependencies. LSTM solves this with the cell state cтВЬ: the gradient flows through cтВЬ via additive update (cтВЬ = fтВЬтКЩcтВЬтВЛтВБ + ...) without being multiplied by weights repeatedly. The forget gate gradient is тИВcтВЬ/тИВcтВЬтВЛтВБ = fтВЬ, which is close to 1 when the gate is open.
Q02What is the difference between LSTM and GRU?
mid+
GRU (Gated Recurrent Unit) simplifies LSTM from 4 gates to 2: Reset gate (rтВЬ = ╧Г(Wr┬╖[hтВЬтВЛтВБ,xтВЬ])) and Update gate (zтВЬ = ╧Г(Wz┬╖[hтВЬтВЛтВБ,xтВЬ])). Hidden state: hтВЬ = (1-zтВЬ)тКЩhтВЬтВЛтВБ + zтВЬтКЩtanh(Wh┬╖[rтВЬтКЩhтВЬтВЛтВБ,xтВЬ]). GRU merges cell and hidden state. GRU: 25% fewer parameters, faster training, similar performance on most tasks. LSTM: slightly better on very long sequences, separate memory cell is more expressive. Practical choice: GRU for quick experiments, LSTM for best results on sequential tasks, Transformer for long contexts.

Q03When would you choose an RNN over a Transformer?
hard+
Despite Transformers dominating NLP, RNNs still win in specific scenarios: (1) Very long sequences with limited compute тАФ RNN is O(n) in memory, Transformer is O(n┬▓); (2) Online/streaming inference тАФ RNNs process one token at a time with constant compute; Transformers need the full sequence; (3) Tiny models тАФ an LSTM with 1K parameters can do useful time-series work; the smallest transformers are >1M; (4) Real-time audio/sensor processing тАФ causal, constant-latency. Modern alternatives: State Space Models (Mamba) combine RNN's O(n) inference with Transformer-like training parallelism.
