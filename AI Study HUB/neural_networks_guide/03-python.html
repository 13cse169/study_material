NUMPY FROM SCRATCH

Build a neural network using only NumPy
🔹Why NumPy First?

Before using frameworks, implement everything from scratch in NumPy. You'll understand exactly what happens under the hood — forward pass, backprop, weight updates — no magic.

Python · Full MLP in NumPy
import numpy as np

class NumpyMLP:
    def __init__(self, dims):
        self.W = [np.random.randn(d1,d2)*np.sqrt(2/d1)
                   for d1,d2 in zip(dims,dims[1:])]
        self.b = [np.zeros(d) for d in dims[1:]]

    def forward(self, x):
        self.cache = [x]
        for i,(W,b) in enumerate(zip(self.W, self.b)):
            x = x @ W + b
            if i < len(self.W)-1: x = np.maximum(0, x)  # ReLU
            self.cache.append(x)
        e = np.exp(x - x.max(-1, keepdims=True))
        return e / e.sum(-1, keepdims=True)

PYTORCH BASICS

Tensors, autograd, nn.Module
🔹PyTorch Core Concepts

PyTorch is the dominant research framework. Its dynamic computation graph (eager execution) makes debugging easy — you can use standard Python debuggers.

Python · PyTorch MLP
import torch
import torch.nn as nn
import torch.nn.functional as F

class MLP(nn.Module):
    def __init__(self, in_dim, hidden, out_dim):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(in_dim, hidden), nn.ReLU(),
            nn.Dropout(0.3),
            nn.Linear(hidden, hidden), nn.ReLU(),
            nn.Linear(hidden, out_dim)
        )
    def forward(self, x): return self.net(x)

model = MLP(784, 256, 10)
criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.AdamW(model.parameters(), lr=3e-4)

TensorFlow 2.x with Keras provides an intuitive API. Use Sequential for simple models, Functional API for complex architectures with multiple inputs/outputs.

Beginner
Q01What is the difference between PyTorch and TensorFlow?
easy+
Execution: PyTorch uses eager execution (define-by-run) — you write Python code that executes immediately, making debugging easy. TensorFlow 1.x used graph execution (define-then-run); TF 2.x defaults to eager. Research vs Production: PyTorch dominates research (papers, HuggingFace, open weights). TensorFlow/Keras is popular in production (TF Serving, TFLite, TF.js). Syntax: PyTorch feels more Pythonic and NumPy-like. Ecosystem: PyTorch has Torchvision, Torchaudio, HuggingFace. TensorFlow has TFHub, TFLite. For new projects: PyTorch unless you have a TensorFlow-specific deployment target.
Q02What is a DataLoader in PyTorch?
easy+
PyTorch's DataLoader wraps a Dataset and provides: (1) Batching — collects samples into tensors of shape (batch_size, ...); (2) Shuffling — randomises order each epoch; (3) Multi-process loading — num_workers>0 loads data in parallel processes while GPU trains; (4) Collation — handles variable-length sequences. You implement __len__ and __getitem__ in your Dataset. The DataLoader then handles all the heavy lifting. Always use num_workers≥2 for image datasets — data loading is often the bottleneck.
Advanced
Q03What is torch.no_grad() and when should you use it?
hard+
torch.no_grad() is a context manager that disables gradient computation. PyTorch normally tracks all tensor operations to build a computation graph for backprop. This tracking adds memory overhead (~2x) and computation. When you're not doing backprop (inference, evaluation), this tracking is wasteful. Always use during: (1) validation loop; (2) inference/prediction; (3) when computing metrics. Incorrect usage causes memory leaks in inference loops. Also note: @torch.no_grad() decorator works on functions. Different from model.eval() which controls behaviour of Dropout/BatchNorm — use both together.
Q04Explain custom loss functions in PyTorch.
hard+
Custom losses are implemented as Python functions or nn.Module subclasses. Key requirement: use PyTorch tensor operations so autograd can backpropagate. Example: def focal_loss(pred, target, gamma=2): p = pred.softmax(-1)[range(len(target)), target]; return -((1-p)**gamma * p.log()).mean(). As nn.Module: subclass and implement forward(). The loss must return a scalar tensor. Avoid: converting to numpy inside the loss (breaks autograd), using Python loops over batch (use vectorised ops), forgetting to handle edge cases (log(0), division by zero).
