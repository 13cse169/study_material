TRAINING LOOP

The standard PyTorch training pattern
Python · Professional Training Loop
import torch
from torch.cuda.amp import autocast, GradScaler

scaler = GradScaler()  # mixed precision

def train_epoch(model, loader, optimizer, criterion, device):
    model.train()
    total_loss, correct, total = 0, 0, 0
    for X, y in loader:
        X, y = X.to(device), y.to(device)
        optimizer.zero_grad(set_to_none=True)  # faster than zero_grad()
        with autocast():  # fp16 forward pass
            logits = model(X)
            loss = criterion(logits, y)
        scaler.scale(loss).backward()
        scaler.unscale_(optimizer)
        torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
        scaler.step(optimizer)
        scaler.update()
        total_loss += loss.item()
        correct += (logits.argmax(1) == y).sum().item()
        total += y.size(0)
    return total_loss/len(loader), correct/total

Q01What is mixed precision training and why use it?
mid+
Mixed precision training uses FP16 (half precision) for most operations and FP32 for critical accumulations. Benefits: (1) 2x memory reduction — fit larger batches; (2) 2-3x faster on Tensor Core GPUs (V100, A100). The GradScaler multiplies the loss by a large scale factor before backward to prevent FP16 underflow (gradients that would become 0). After backward, it unscales before the optimizer step. PyTorch: torch.cuda.amp.autocast() in forward pass + GradScaler for backward. Modern GPUs (A100, H100) support BF16 which has same range as FP32 — often better than FP16.
Q02What is learning rate warmup and why is it used?
mid+
Warmup gradually increases the learning rate from 0 (or a small value) to the target LR over the first N steps/epochs. Rationale: at the start of training, parameters are random and gradients are noisy and large. A large LR immediately can cause divergence. Warmup lets the optimiser's momentum terms stabilise before committing to large steps. Especially important for: Transformers (warm up for 4% of total steps), fine-tuning (avoids catastrophic forgetting), large batch training. After warmup: cosine annealing or linear decay to 0. Common schedule: warmup 500 steps → cosine to step 10,000.
Advanced
Q03Explain gradient accumulation and when to use it.
hard+
Gradient accumulation simulates a larger batch size by accumulating gradients over multiple forward/backward passes before doing a parameter update. Example: effective batch 256 with GPU that can only fit 32 → accumulate 8 steps. Code: skip optimizer.step() and zero_grad() for 7 steps, execute on step 8. Important: loss must be divided by accumulation steps (loss = loss / grad_accum_steps) before backward. Use with mixed precision carefully — scaler.update() only on real steps. When to use: training large models (>1B params), memory-constrained hardware, trying to match published large-batch training configs.
