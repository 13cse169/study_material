SELF-ATTENTION

The core of every modern language model
🔹Scaled Dot-Product Attention

Every token attends to every other token simultaneously. Q, K, V projections of the input let the model selectively aggregate information.

Attention(Q,K,V) = softmax(QKᵀ/√dₖ)·V MultiHead = Concat(head₁...headₕ)·Wᴼ
Python · Multi-Head Attention from Scratch
import torch, math
import torch.nn as nn

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model=512, heads=8):
        super().__init__()
        self.h, self.d = heads, d_model//heads
        self.qkv = nn.Linear(d_model, d_model*3)
        self.out = nn.Linear(d_model, d_model)

    def forward(self, x, mask=None):
        B,T,C = x.shape
        q,k,v = self.qkv(x).chunk(3,dim=-1)
        q,k,v = [t.view(B,T,self.h,self.d).transpose(1,2) for t in (q,k,v)]
        sc = (q@k.transpose(-2,-1))/math.sqrt(self.d)
        if mask is not None: sc = sc.masked_fill(mask==0,-1e9)
        out = (sc.softmax(-1)@v).transpose(1,2).contiguous().view(B,T,-1)
        return self.out(out)

FINE-TUNING & LoRA

Adapt pretrained models to custom tasks efficiently
🔹LoRA — Low-Rank Adaptation
W = W₀ + BA where B∈ℝ^(d×r), A∈ℝ^(r×k), r << min(d,k) Only B and A are trained — 0.1-1% of original parameters
Python · QLoRA Fine-tuning
from peft import LoraConfig, get_peft_model
from transformers import AutoModelForCausalLM, BitsAndBytesConfig
import torch

bnb = BitsAndBytesConfig(load_in_4bit=True,
    bnb_4bit_quant_type='nf4', bnb_4bit_compute_dtype=torch.bfloat16)

model = AutoModelForCausalLM.from_pretrained(
    'meta-llama/Llama-3.2-3B', quantization_config=bnb, device_map='auto')

config = LoraConfig(r=16, lora_alpha=32,
    target_modules=['q_proj','v_proj'], task_type='CAUSAL_LM')
model = get_peft_model(model, config)
model.print_trainable_parameters()  # ~0.1%

INTERVIEW Q&A

Transformers · 10 Questions
Intermediate
Q01Why do we scale by √dₖ in attention?
mid+
For large dₖ, dot products QKᵀ grow in magnitude (variance scales with dₖ if Q,K have unit variance). Large values push softmax into saturation regions where gradients are tiny. Scaling by 1/√dₖ keeps variance ≈1 regardless of dimension, stabilising gradients through softmax. Example: dₖ=64 → scale by 1/8.
Q02What is the difference between encoder-only, decoder-only, and encoder-decoder models?
mid+
Encoder-only (BERT): Bidirectional attention, sees full sequence. Best for understanding: classification, NER, QA. Decoder-only (GPT): Causal attention, sees only past tokens. Best for generation: completion, chat, code. Encoder-Decoder (T5, BART): Encoder reads input bidirectionally, decoder generates output attending to encoder. Best for seq2seq: translation, summarisation. Rule of thumb: encoder for discriminative tasks, decoder for generative tasks, enc-dec for transformation tasks.

Q04What is Flash Attention and why is it important?
hard+
Standard attention materialises the full n×n attention matrix in HBM memory — O(n²) space. For n=8192 tokens: 8192² × 2 bytes = 128MB per head per layer. FlashAttention rewrites the CUDA kernel to tile computation in SRAM: processes blocks of Q,K,V that fit in fast on-chip memory, uses online softmax normalisation (log-sum-exp trick) to avoid materialising full matrix. Result: O(n) HBM reads/writes, 2-4x wall-clock speedup, enables much longer context lengths. Flash Attention 2/3 add further optimisations (work partitioning, warp specialisation).
